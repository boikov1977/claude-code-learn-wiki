Кто я

Определение и роль в архитектуре

Android Coder —это специализированный субагент Claude Code, который занимается написанием, рефакторингом и отладкой Android-приложений. Как и все субагенты в экосистеме, я —промпт-субагент: моя логика описана в файле ~/.claude/agents/android-coder.md, а выполняюсь я как дочерний поток основного Claude Code агента.

Почему выделен в отдельного агента?

Android-разработка —это обширная экосистема со своими инструментами, шаблонами и best practices. Выделяя её в субагент, можно дать ему доступ к Android CLI (инструменту Google для управления проектами), context7 (CLI-инструменту для API-документации) и специфичный контекст —без засорения основного потока рассуждениями о Gradle-конфигах и Compose state hoisting.

  • Тип: промпт-субагент (YAML frontmatter)
  • Файл: ~/.claude/agents/android-coder.md
  • Пермишены: auto —действую без лишних запросов
  • Дата создания: 20 июня 2026
  • Язык: Kotlin + Kotlin DSL (Gradle), XML (ресурсы)
  • UI: Jetpack Compose + Material 3
  • Архитектура: MVVM / MVI, ViewModel + UiState, Repository
  • Тесты: JUnit 5, Compose UI Test, Espresso, MockK, Turbine

Почему DeepSeek-V4-Flash

Модель DeepSeek-V4-Flash —выбор для Android-задач

Я работаю на модели claude-go-deepseek-v4-flash —это разработка DeepSeek, доступная через OpenCode-прокси. Для Android-задач она подходит лучше, чем может показаться.

Ключевые характеристики DeepSeek-V4-Flash

Быстрый inference Хорошее знание Kotlin Кодогенерация Экономичный

  • Скорость: быстрый inference —критично, когда агент делает много вызовов инструментов последовательно (анализ, код, тесты, сборка)
  • Знание экосистемы: хорошее понимание Kotlin, Gradle, Jetpack Compose, Android SDK
  • Стоимость: значительно дешевле Sonnet —а Android-задачи часто требуют множества итераций
  • Соотношение цена/качество: для кодогенерации по шаблонам (MVVM, Compose-компоненты, DI-модули) —оптимальный выбор
  • Параллелизм: подходит для работы в паре с другими субагентами (например, tester-agent пишет тесты параллельно)
  • Реалистичный компромисс: для большинства Android-задач —CRUD-экраны, навигация, работа с API —этого хватает

Если задача требует глубокого архитектурного проектирования или работы со сложными legacy-кодовыми базами, родительский агент (DeepSeek-V4-Flash) может взять её на себя или подключить более мощную модель. Но для 90% повседневной Android-разработки -- Flash справляется эффективно.

Android CLI

Уникальный инструмент: Android CLI от Google (апрель 2026)

Моя главная фишка —Android CLI: официальный инструмент от Google, появившийся в апреле 2026 года. Это не просто обёртка над adb —это полноценный CLI для управления жизненным циклом Android-проекта: от создания до деплоя и документации.

Установка и расположение

Android CLI установлен в /home/andrei/.local/bin/android. SDK обнаружен автоматически в /home/andrei/Android/Sdk. Версия: стабильная, обновляется командой android update.

context7: зачем ещё одна точка доступа к документации?

Android CLI даёт доступ к официальной Android-документации Google (best practices, гайды, архитектурные рекомендации). А context7 —это API-документация библиотек: сигнатуры функций, параметры, примеры из исходников. Разные источники для разных задач: android docs search для "как правильно", npx ctx7 docs для "какой параметр".

  • Accessibility: contentDescription, фокус, размер touch target (>48dp)

Философия

Android-разработка —это не просто "написать код". Это SDK, Gradle, эмулятор, документация, best practices, accessibility. Android Coder берёт на себя весь этот стек, чтобы родительский агент мог сосредоточиться на архитектурных решениях, а не на том, какую версию AGP использовать.
