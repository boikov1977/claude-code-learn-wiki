Полный обзор 7 агентов нашей экосистемы: кто они, зачем нужны и как работают вместе

Зачем вообще субагенты

Концепция, которая стоит за разделением труда

Субагент — это отдельный специалист внутри Claude Code, который выполняет узкую задачу со своим контекстом и при необходимости своей моделью. Основной агент (тот, с кем ты общаешься) вызывает его по команде, передаёт задачу и получает результат.

Почему не делать всё в одном агенте? Потому что это:

  • Чистый контекст — агент-писатель не видит твои архитектурные дебаты, архитекторы — не засоряют контекст кодом
  • Своя модель — документацию можно писать дешёвой моделью, а архитектуру — мощной
  • Свои инструменты — explorer только читает, writer только документирует, openspec-manager управляет скиллами
  • Параллельность — три архитектора могут работать одновременно над одним вопросом
  • Специализация — каждый агент глубоко знает свою область и делает её лучше универсала

Формат агента

Каждый субагент — это Markdown-файл в ~/.claude/agents/ с YAML-фронтматтером (имя, модель, инструменты, MCP-серверы) и промптом-инструкцией в теле. Это промпт-субагенты, а не отдельные процессы: они выполняются как дочерний поток основного Claude Code.

Каталог агентов

Подробно о каждом: роль, модель, инструменты, когда вызывать

🧠
openspec-manager
Spec-driven разработка
Управляет полным lifecycle изменений по методологии OpenSpec: proposal, design, tasks, реализация, синхронизация спек, архивация. Самый «рабочий» агент — с 11 скиллами, изолированным MCP-сервером и собственной файловой структурой.
model: inherit MCP: openspec-mcp (uvx) 11 skills Read, Write, Edit, Bash, Glob, Skill
Когда вызывать: начало новой фичи, рефакторинг, архитектурное изменение — любая структурированная работа, где нужны спецификации и план перед кодом.
✍️
writer-agent
Документация
Анализирует изменения в коде и создаёт/обновляет документацию: README, CHANGELOG, docs/ и HTML-страницы для вики. Использует дешёвую модель MiMo-V2.5 с максимальным усилием (xhigh) — дёшево, но старательно. Умеет искать информацию в интернете через poisk-mcp.
model: claude-go-mimo-v2.5 effort: xhigh MCP: poisk-mcp skill: deep-research Read, Write, Edit, Bash, Glob, Grep
Когда вызывать: нужно описать проект, обновить документацию, создать HTML-страницу в вики, написать CHANGELOG, сгенерировать README для нового модуля.
🔍
explorer
Быстрый поиск кода
Read-only агент для навигации по кодовой базе. Ищет файлы, анализирует структуру, отвечает на вопросы «где используется Y», «как устроен X». Ничего не меняет — только читает и ищет.
model: inherit Read, Glob, Grep, Bash
Когда вызывать: найти где используется функция, понять структуру модуля, проверить зависимости.
🖥️
statusline-setup
Настройка статус-строки
Маленький специализированный агент: настраивает статус-строку Claude Code через /statusline. Читает текущие настройки, спрашивает что видеть, записывает конфиг в settings.json.
model: inherit Read, Write, Edit, Bash, Glob
Когда вызывать: настроить/изменить/убрать статус-строку, показать ветку, время, погоду.
🚀
innovator
Архитектор-инноватор
Смелый архитектор, смотрящий на 2–3 шага вперёд. Предлагает передовые решения, новые технологии, не боится ломать устаревшие подходы. Имеет доступ к Context7 для актуальной документации библиотек.
model: claude-go-minimax-m3 MCP: context7 (remote) Read, Write, Edit, Bash, Glob, Grep
Когда вызывать: нужна самая передовая идея, радиальная переработка, новый стек.
⚖️
pragmatist
Архитектор-прагматик
Ищет золотую середину. Предлагает реалистичные, внедряемые решения с учётом существующего кода, сроков и стоимости. Мастер сравнительных таблиц и cost-benefit анализа. Самая мощная модель в Triage.
model: claude-go-deepseek-v4-pro MCP: context7 (remote) Read, Write, Edit, Bash, Glob, Grep
Когда вызывать: нужно оптимальное, реалистичное решение с оценкой сроков и стоимости.
🛡️
conservative
Архитектор-консерватор
Хранитель стабильности. Отстаивает проверенные решения, указывает на риски, нестабильность и подводные камни. Не блокирует — а предлагает безопасные альтернативы. Каждый риск оценивает по severity.
model: claude-go-qwen3.7-plus MCP: context7 (remote) Read, Write, Edit, Bash, Glob, Grep
Когда вызывать: оценить риски, проверить безопасность, понять что может пойти не так.

Архитектурный отдел (Triage)

Процесс из 2 раундов, где три агента обсуждают решение, а затем синтезируется вердикт

Когда перед основным агентом встаёт архитектурный вопрос («Какую БД выбрать?», «Стоит ли менять стек?»), он запускает Triage — структурированный процесс из двух раундов, в котором участвуют три архитектора с разными взглядами.

🚀 Инноватор

Смелые решения, новые технологии, 2–3 шага вперёд

⚖️ Прагматик

Баланс, cost-benefit, реалистичные сроки

🛡️ Консерватор

Риски, стабильность, проверенные подходы

Как это работает

Всё общение идёт через файлы в папке .triage/<task-name>/. Каждый агент пишет свой файл, потом читает чужие и пишет rebuttal (ответ). Финальный вердикт синтезирует writer-agent.

Два раунда

Раунд 1: Proposal

Каждый архитектор пишет свою позицию (до 300 слов): название подхода, технологии, плюсы и минусы. Файлы: 01-innovator.md, 02-pragmatist.md, 03-conservative.md.

Раунд 2: Rebuttal

Каждый читает чужие позиции и пишет ответ (до 200 слов): с чем согласен, с чем нет, свой скорректированный вариант. Файлы: *-rebuttal.md.

Экономика

Все три архитектора в Triage используют дешёвые модели (minimax-m3, qwen3.7-plus) или среднюю (deepseek-v4-pro). Документация — на дешёвой MiMo с высоким усилием. Простые агенты (explorer, statusline, openspec-manager) — inherit, без дополнительных затрат. Это позволяет держать расходы низкими при высоком качестве.

MCP-серверы в экосистеме

Какие внешние сервисы подключены к каким агентам

MCP-сервер Тип Агент Назначение
openspec-mcp Изолированный (uvx) openspec-manager Быстрые операции с OpenSpec: список изменений, чтение спек, валидация
poisk-mcp Локальный writer-agent Поиск в интернете: search_web, browse_summarize, read_webpage
context7 Remote (облачный) innovator, pragmatist, conservative Актуальная документация библиотек через npx ctx7 docs

Изолированный MCP — объявлен в конфиге агента, доступен только ему. Глобальный MCP (poisk-mcp) — добавлен через claude mcp add, но подключён только к writer-agent. Remote MCP (context7) — облачный сервис с API-ключом в заголовках.
