Worktree в Claude Code

Изолированные копии проекта для безопасных экспериментов — всё просто, как по маслу

🤔 Что такое Worktree (рабочее дерево)?

Worktree — это изолированная копия твоего проекта, в которой можно спокойно ломать, менять и экспериментировать, не боясь испортить основную папку.

💡 Аналогия на пальцах: Представь, что у тебя есть комната с рабочим столом (основной проект). Worktree — это снять такую же копию комнаты в отдельном кабинете. Ты можешь разбросать там всё что хочешь, сжечь стол, перевернуть стул — а в основной комнате ноль изменений. Чисто и безопасно.

Технически это git worktree — Git умеет держать несколько рабочих директорий, привязанных к разным веткам. Claude Code просто оборачивает это в удобную команду:

// Claude Code создаёт worktree автоматически
EnterWorktree → создаёт ветку + копию проекта

// Все изменения летят только туда
// Основной проект — нетронут

ExitWorktree → выход (сохранить или удалить)

🎯 Зачем это нужно?

🧪
Эксперименты
Рискованный рефакторинг? Попробуй в worktree — не получится — удалил и всё
🔀
Параллельная работа
Основной код стабилен, а ты в worktree тестируешь новую фичу
🤖
Субагенты
Запуск агента с isolation: worktree — ему своя копия репозитория
🔍
Код-ревью
Worktree на каждую ветку PR — смотришь изменения изолированно

💡 Проще говоря: Worktree — это как второй монитор для того же компьютера. Оба монитора используют одну «железку» (Git), но работают в своих окнах. Закрыл одно окно — другое работает как ни в чём не бывало.

💻 Как это выглядит в Claude Code

Ты просто говоришь Claude Code создать worktree — он делает всё сам:

Создание worktree
# Ты говоришь Claude Code:
> Создай worktree для экспериментов с новым API

# Claude Code выполняет EnterWorktree и создаёт:
.claude/worktrees/experiment-new-api/

# Ветка: experiment-new-api (или как назовёшь)
# Рабочая директория переключена
# Все команды идут в worktree
Работа внутри worktree
# Работаешь как обычно — Claude Code уже в worktree
> Рефактори модуль авторизации
> Напиши тесты для нового эндпоинта
> Запусти линтер

# Все изменения — только в .claude/worktrees/experiment-new-api/
# Основная папка — чистая как слеза 👌
Выход из worktree
# Готов, хочешь выйти:
> Выйди из worktree

# Claude Code спросит action:
action: keep # — worktree остаётся на диске, можно вернуться
action: delete # — worktree удаляется, ветка остаётся в Git

# Если в worktree есть незакоммиченные изменения —
# инструмент не даст удалить без подтверждения! Безопасно.

📋 Когда использовать Worktree

Ситуация Worktree? Почему
🧪 Начинаешь большую фичу Да! Изолируешь эксперимент, не мешаешь коллегам
🤖 Запускаешь субагента Да! isolation: worktree — агент получает свою копию
😰 Боишься что-то сломать Да! Ничего не удалишь в основной папке
📊 Работаешь над стабильным кодом Да! Не рискуешь стабильной базой
🔍 Ревьюешь PR Полезно Отдельная копия для каждой ветки
✏️ Мелкий фикс (1 файл) Не обязательно Работай в основной — проще

⚠️ Ограничения и подводные камни

🔧
Не все инструменты — некоторые tool'ы и хуки могут некорректно работать в worktree. Если что-то странное — выйди из worktree и проверь в основной директории.
🏠
Возврат домой — после ExitWorktree сессия Claude Code возвращается в обычную директорию. Ты как будто перешёл из кабинета обратно в свою комнату.
💾
При action: keep — worktree остаётся на диске. Не забывай удалять старые, иначе будут занимать место. Команда git worktree list покажет все worktrees.
🔒
Безопасность удаления — если в worktree есть несохранённые изменения, инструмент не даст удалить без подтверждения. Всегда можно вернуться.

💡 Полезные команды Git

Для понимания что происходит «под капотом»:

# Посмотреть все worktrees
$ git worktree list

# Результат:
/home/user/my-project abc1234 [main]
/home/user/my-project/.claude/worktrees/feature-x def5678 [feature-x]

# Ручное удаление worktree (если Claude Code не помог)
$ git worktree remove .claude/worktrees/feature-x

# Убрать «мёртвые» worktree (если папка удалена вручную)
$ git worktree prune

✨ Итого

Worktree = БЕЗОПАСНАЯ КОПИЯ ПРОЕКТА.

🏠 Основной проект — чистый и нетронутый
🌳 Worktree — твоя песочница для экспериментов
🔄 EnterWorktree → работаешь → ExitWorktree
🛡️ Ничего не потеряешь и не сломаешь

Если боишься — делай в worktree. Не боишься — всё равно делай в worktree. 😄
