🧩 Что такое скиллы?

Скиллы — это готовые инструкции, которые Claude Code умеет выполнять. Они вызываются через /имя-скилла прямо в чате и работают как встроенные команды.

Проще простого: Скиллы — это не тулы и не расширения. Это промпты, которые говорят Claude: «сделай то-то, вот тебе инструкция». Они уже встроены в Claude Code — ничего ставить не надо. Просто напиши /code-review и готово.

Есть два типа скиллов:

  • базовые — от Anthropic, встроены «из коробки»
  • кастомные — пишем сами, кладём в ~/.claude/skills/

Все базовые скиллы описаны в таблице ниже. Кастомные — отдельная тема (про OpenSpec расскажем в конце).

🔍 Код и ревью

Вся работа с кодом: проверка, ревью, упрощение, аудит безопасности. Основные «рабочие лошадки».

  • /code-review — быстрый, на уровне диффа
  • /review — глубокий, с контекстом всего PR
  • /simplify — причесывает код после написания
  • /security-review — находит уязвимости в чувствительном коде

▶️ Запуск и проверка

Всё, что связано с запуском приложения и проверкой результатов:

  • /run — запускает проект (сам определяет команду)
  • /verify — проверяет, что фикс реально работает
  • /run-skill-generator — обучает /run твоему проекту (разовый setup)

🎓 Поиск и исследования

Когда нужна информация извне:

  • /deep-research — глубокий поиск с верификацией источников
  • /claude-api — быстрая справка по API Claude

📦 Утилиты

Помощники для рутинных задач:

  • /batch — пакетная обработка (много файлов за раз)
  • /debug — анализ ошибок и дебаг
  • /loop — повторение по расписанию

⚙️ Настройка

Настройка самого Claude Code и его поведения:

  • /init — генерация CLAUDE.md для нового проекта
  • /update-config — редактирование settings.json
  • /fewer-permission-prompts — меньше запросов «можно?»
  • /keybindings-help — настройка горячих клавиш

⚔️ /code-review vs /review — в чём разница?

Это два самых частых скилла, и их легко путают. Вот чёткое сравнение:

/code-review 🔍 /review 👁️
Что смотрит Текущий diff (незакоммиченные изменения) Весь PR целиком + контекст проекта
Глубина Быстрый взгляд — баги, стиль, очевидные проблемы Полноценное ревью — архитектура, паттерны, тесты
GitHub Нет — только пишет в чат Да — может постить комментарии прямо в PR
Когда Перед коммитом, «погляди на мой код» Готов отправить PR, нужен формальный разбор
Аналогия Коллега бегло смотрит экран Формальный Code Review на GitHub
Простое правило:
🔍 /code-review = «глянь мой дифф перед коммитом»
👁️ /review = «сделай полное ревью PR с комментариями в GitHub»

🔧 Наши скиллы (OpenSpec)

Помимо базовых, у нас есть кастомные скиллы — OpenSpec. Они живут в ~/.claude/skills/ и работают по специальной методологии spec-driven разработки.

OpenSpec скиллы

Это отдельная экосистема для управления процессом разработки через спеки:

  • openspec-propose — предложить изменение
  • openspec-apply — применить спеку
  • openspec-sync — синхронизировать
  • openspec-archive — архивировать завершённое
  • openspec-explore — исследовать кодовую базу
  • openspec-ff — feature flag управление
  • openspec-new — начать новый сценарий
  • openspec-continue — продолжить работу
  • openspec-verify — верифицировать результат
  • openspec-bulk-archive — массовый архив
  • openspec-onboard — онбординг нового разработчика

Подробно про OpenSpec — в отдельной статье. Здесь просто знай, что они существуют.

💡 Как отличить базовый от кастомного: Набери / в чате — автодополнение покажет все доступные скиллы. Базовые помечены как built-in, кастомные — по имени файла из ~/.claude/skills/.

🎯 Лайфхаки по использованию

1. Комбинация скиллов для полного цикла

Идеальный workflow при работе с фичей:

# 1. Написал код /verify → убедился что работает # 2. Почистил /simplify → убрал дубли, причесал # 3. Проверил /code-review → быстрый взгляд перед коммитом # 4. Для важного кода /security-review → если там auth, payments, API keys # 5. Коммит и PR /review → полноценное ревью PR

2. /debug — первый друг при ошибке

Когда тесты упали или пошло что-то странное — не пытайся разобраться сам. Скинь ошибку Claude через /debug. Он проанализирует стектрейс, найдёт связанные файлы и предложит фикс. Это быстрее, чем гуглить.

3. /batch для миграций

Нужно поменять API-вызов в 50 файлах? /batch сделает это за раз. Опиши паттерн «найти все вызовы oldApi() и заменить на newApi()» — и Claude обработает всё.

4. /run-skill-generator — настрой один раз

Если /run не угадал, как запускать твой проект — вызови /run-skill-generator. Он создаст инструкцию, и потом /run будет знать как запускать именно твой проект.

⚡ Автодополнение — твой друг. Просто набери / в чате и Claude покажет все доступные скиллы. Не надо запоминать названия — достаточно начать вводить и выбрать из списка.

Итого: 16 базовых скиллов — от быстрого /code-review до глубокого /deep-research. Они уже встроены, ничего ставить не надо. Просто набери / и выбери нужный. 🚀
