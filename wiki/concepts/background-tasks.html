🔄 Что такое Background Tasks?

Представь: ты готовишь ужин и одновременно слушаешь подкаст. Руки заняты едой, а уши — подкастом. И то, и другое происходит параллельно, одно другому не мешает.

Background Tasks — это запуск задач в фоне. Пока Claude работает над твоим текущим запросом, в фоне может параллельно выполняться другая задача. Ты не ждёшь, пока одна закончится, чтобы начать другую.

Проще простого: Background Tasks — это возможность запустить Claude на одну задачу, а самому продолжить работать с другим Claude. Как открыть вторую вкладку в браузере — одна страница грузится, пока ты смотришь другую.

1. Флаг --bg при старте

Запускаешь Claude сразу в фоновом режиме. Он уходит в фон и работает — ты пока занимаешься своим делом.

# Запустить исследование бага в фоне claude --bg "исследуй баг #123 в модуле auth" # Анализ большого диффа в фоне claude --bg "проанализируй git diff HEAD~10 и составь отчёт" # Пока фоновая задача работает — можно открыть вторую сессию claude # обычный режим, работаем параллельно

2. Ctrl+B в сессии

Сидишь в сессии, понимаешь что задача долгая — отправляешь её в фон одной комбинацией клавиш.

Сценарий: Ты пишешь Claude длинный промпт. Вместо того чтобы ждать ответ 5 минут — нажимаешь Ctrl+B. Задача уходит в фон, а ты свободен для новых запросов в этой же сессии.

3. Через субагента

Агент (например, через /agents или из конфига) может быть запущен в фоне по умолчанию. Это основа для мульти-агентных сценариев.

🎯 Когда это нужно

🔍 Параллельное исследование

Два агента ищут баг в разных местах кода. Один проверяет backend, другой — frontend. Результаты обаотчёт (сообщат) когда будут готовы.

📊 Долгая задача

Разбор логов на 50к строк, анализ большого диффа, рефакторинг целого модуля. Не хочется ждать — отправляем в фон и работаем дальше.

✅ Фоновая проверка

Пока ты пишешь код, в фоне агент проверяет CI, запускает тесты, линтит. Как только что-то упадёт — сразу узнаешь.

📝 Генерация документации

Пока ты работаешь над фичей, фоновый агент генерирует доку, обновляет CHANGELOG. Код и документация идут параллельно.

🚫 Ограничения

Background Tasks — мощная штука, но есть важные нюансы, которые надо знать:

Что нужно помнить

Ограничение Что значит
Разрешения при старте Если при запуске фоновой задачи ты не выдал разрешение на инструмент — задача будет действовать в режиме автозапрета. Не сможет писать файлы, запускать команды и т.д. Всё важное — решай при запуске!
Не восстанавливается При --resume фоновые задачи не поднимаются заново. Если сессия упала — фоновые задачи пропали.
Время жизни Фоновые задачи живут пока жива сессия. Закрыл терминал — всё завершилось.
Ресурсы Каждая фоновая задача — это отдельный контекст. Чем больше задач параллельно, тем больше токенов и тем дороже по API.
💡 Важно: При запуске фоновой задачи Claude спросит разрешения на нужные инструменты. Если ты пропустил этот момент — задача будет работать в усеченном режиме. Лучше сразу выдать все нужные пермишены!

📋 Управление фоновыми задачами

Два способа посмотреть, что происходит в фоне:

Команда Что показывает
/tasks Список всех фоновых задач — статус, описание, время запуска
/bg Панель управления: текущие задачи, возможность отменить или посмотреть результат

Типичный workflow

  1. Запустил фоновую задачу через --bg или Ctrl+B
  2. Продолжил работать в основной сессии
  3. Через какое-то время написал /tasks — посмотрел статус
  4. Задача готова — результат появился в списке
  5. Прочитал результат, использовал информацию

🏗️ Связь с Workflow

Background Tasks — это строительный блок для более сложных сценариев. Когда ты комбинируешь несколько фоновых задач с основной сессией — получаешь параллельный мульти-агентный workflow.

Пример: разработка фичи

# Терминал 1: основная работа claude # пишешь код фичи # Терминал 2: фоновая проверка тестов claude --bg "запусти все тесты и сообщи что упало" # Терминал 3: фоновый анализ зависимостей claude --bg "проверь не устарели ли зависимости в package.json" # Итого: пока ты кодишь — три параллельных потока работают на тебя

Преимущества параллельного подхода

  • Скорость: Вместо последовательного ожидания — всё идёт параллельно
  • Покрытие: Разные агенты проверяют разные аспекты одновременно
  • Фокус: Ты не отвлекаешься на второстепенные задачи — они в фоне
  • Автоматизация: Меньше ручных шагов — агенты сами формируют отчёт

Итого: Background Tasks — это способ запускать задачи параллельно, не блокируя основную сессию. Используй --bg для запуска, Ctrl+B для отправки в фон, /tasks для проверки статуса. Помни про ограничения: пермишены решаются при старте, а при падении сессии фоновые задачи не восстанавливаются. 🔄
