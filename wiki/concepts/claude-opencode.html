🤖 Что такое claude-opencode?

Claude Code — это CLI-инструмент от Anthropic. Он умеет читать файлы, писать код, запускать команды, общаться с MCP-серверами. Но вот проблема: для работы ему нужна модель Claude (Haiku, Sonnet, Opus), а для этого — подписка Claude Pro или Team. Если подписки нет — Claude Code молчит.

claude-opencode — это система из двух компонентов (лаунчер + прокси), которая позволяет обмануть Claude Code. Ты поднимаешь локальный прокси-сервер на порту 4000, Claude Code думает, что общается с Anthropic API, а на самом деле запросы уходят на OpenCode Go — сервис, который даёт доступ к моделям DeepSeek, Qwen, MiniMax, GLM, Kimi, MiMo и другим.

Проще простого: claude-opencode — это «переходник» между Claude Code и китайскими моделями. Ты общаешься с Claude Code как обычно, он думает что работает с Claude, а на самом деле запросы обрабатываются DeepSeek V4, Qwen 3.x или MiniMax M3. Подписка на Claude не нужна.

💰 Зачем это нужно

💰 Экономия

Не нужна подписка Claude Pro ($20/мес) или Team ($25/мес). OpenCode Go предоставляет доступ к моделям по другому тарифу — часто значительно дешевле.

🇨🇳 Китайские модели

DeepSeek V4, Qwen 3.x, MiniMax M3, GLM 5, Kimi K2.7 — модели, которые по качеству уже конкурируют с Claude, а по доступности — значительно выигрывают.

🔧 Гибкость

Переключение модели — одна переменная окружения. Сегодня DeepSeek, завтра Qwen, послезавтра MiniMax. Claude Code не замечает подмены.

🔄 Совместимость

Прокси конвертирует Anthropic Messages API в OpenAI Chat Completions. Любой OpenAI-совместимый API можно подключить — не только OpenCode Go.

Ключевой трюк: трансляция имён моделей

Claude Code отправляет имя модели вроде claude-go-deepseek-v4-flash. Прокси знает, что это значит — он отрезает префикс claude-go- и отправляет deepseek-v4-flash в OpenCode Go. Для Claude Code это выглядит как валидная модель Anthropic.

🧩 Компоненты

Компонент Тип Что делает
claude-opencode Bash-скрипт Лаунчер: поднимает прокси в фоне, стартует Claude CLI, завершает прокси при выходе
anthropic-proxy.py Python / FastAPI Прокси-сервер: конвертирует Anthropic Messages API в OpenAI Chat Completions, транслирует имена моделей, обрабатывает streaming и ошибки
config.sh Bash Конфигурация: API-ключ OpenCode Go, порт прокси, модель по умолчанию
requirements.txt Python Зависимости: FastAPI, uvicorn, httpx

Технические возможности прокси

  • Streaming (SSE) — поддерживает серверные события, как и настоящий Anthropic API
  • Non-streaming — обычные запрос-ответ
  • Системные промпты — конвертирует system из Anthropic-формата в OpenAI
  • Tool calls — конвертирует определения инструментов и их вызовы
  • Изображения — передаёт base64-изображения в content blocks
  • Обработка ошибок — ловит rate limits, таймауты upstream и возвращает в Anthropic-формате
  • Карта моделей — GET /v1/models отдаёт список доступных моделей; Claude Code сам обнаруживает их

Пример: что видит Claude Code

# Claude Code думает, что работает с Anthropic API: POST http://localhost:4000/v1/messages { "model": "claude-go-deepseek-v4-flash", "max_tokens": 4096, "messages": [ {"role": "user", "content": "Привет!"} ] } # А прокси отправляет в OpenCode Go: POST https://opencode.ai/zen/go/v1/chat/completions { "model": "deepseek-v4-flash", "max_tokens": 4096, "messages": [ {"role": "user", "content": "Привет!"} ] }

🛠️ Установка на новой машине

Установка — пять шагов. Нужен Python 3.9+, git и API-ключ OpenCode Go.

Шаг 1. Клонировать репозиторий

git clone https://github.com/boikov1977/claude-opencode.git cd claude-opencode

Шаг 2. Создать виртуальное окружение и установить зависимости

python3 -m venv venv source venv/bin/activate pip install -r requirements.txt

Шаг 3. Настроить API-ключ

# Отредактируй config.sh и вставь свой API-ключ vim config.sh # Содержимое: OPENCODE_API_KEY="твой-ключ-здесь" PROXY_PORT=4000 DEFAULT_MODEL="claude-go-deepseek-v4-flash"

Шаг 4. Создать symlink для удобного запуска

# Чтобы запускать claude-opencode из любой директории ln -s /path/to/claude-opencode/claude-opencode.sh ~/.local/bin/claude-opencode # Убедись, что ~/.local/bin в PATH export PATH="$HOME/.local/bin:$PATH"

Шаг 5. Запустить

# Просто запусти — прокси поднимется автоматически claude-opencode # Или с указанием модели DEFAULT_MODEL="claude-go-qwen3.7-plus" claude-opencode

Что происходит при запуске

  1. Скрипт читает config.sh и получает API-ключ
  2. Запускает anthropic-proxy.py в фоне на порту 4000
  3. Ждёт секунду, пока прокси поднимется
  4. Запускает claude с переменной ANTHROPIC_BASE_URL=http://localhost:4000
  5. При выходе из Claude Code завершает прокси

💬 Использование

После запуска всё работает точно так же, как обычный Claude Code. Единственное отличие — ты выбираешь модель из списка доступных в OpenCode Go, а не из Anthropic.

Переключение моделей

# Модель по умолчанию задаётся в config.sh DEFAULT_MODEL="claude-go-deepseek-v4-flash" # Можно переопределить при запуске DEFAULT_MODEL="claude-go-minimax-m3" claude-opencode # Или в .claude/settings.local.json для основной модели: { "model": "claude-go-deepseek-v4-flash" }

Примеры использования

# Быстрое код-ревью через DeepSeek claude-opencode > "Проверь модуль auth.py на ошибки" # Документирование через MiMo DEFAULT_MODEL="claude-go-mimo-v2.5" claude-opencode > "Напиши README для этого проекта" # Архитектурный анализ через DeepSeek Pro DEFAULT_MODEL="claude-go-deepseek-v4-pro" claude-opencode > "Спроектируй систему аутентификации"

Как это работает

Когда ты вызываешь субагента, Claude Code читает его frontmatter и отправляет запрос с указанным именем модели. Прокси перехватывает это имя, транслирует и отправляет в OpenCode Go. Каждый субагент может работать на своей модели:

  • writer-agent (MiMo V2.5) — хорошо генерирует текст и документацию
  • innovator (MiniMax M3) — сильные рассуждения для архитектуры
  • pragmatist (DeepSeek V4 Pro) — баланс качества и скорости
  • conservative (Qwen 3.7 Plus) — стабильная модель для проверки
  • explorer, openspec-manager (inherit) — берут модель основного Claude

Пример frontmatter агента

--- name: writer-agent model: claude-go-mimo-v2.5 effort: xhigh --- Ты — документирующий агент. Твоя задача — на основе изменений в коде создавать и обновлять документацию.

Прокси увидит claude-go-mimo-v2.5, конвертирует в mimo-v2.5 и отправит в OpenCode Go. Claude Code не замечает подмены.
