💡 Что такое кастомный скилл

Скилл — это готовая инструкция для Claude Code. Ты пишешь в файле SKILL.md: что делать, какие инструменты использовать, какой результат ожидать. Claude выполняет эту инструкцию, когда ты вызываешь скилл через /имя-скилла.

Проще простого: Скилл — это промпт, который ты сохранил в файле и можешь вызывать по имени. Вместо того чтобы каждый раз писать длинную инструкцию — ты пишешь её один раз и вызываешь одной командой.

📁 Структура файла SKILL.md

Каждый скилл — это один файл SKILL.md с двумя частями:

--- name: my-skill description: Краткое описание что делает скилл --- # Инструкции для Claude Здесь ты пишешь что именно Claude должен делать при вызове этого скилла. Подробно, конкретно, с примерами и ограничениями.

Frontmatter (между ---) — это YAML-конфиг. Тело после второго --- — это инструкции.

⚙️ Frontmatter: обязательные и опциональные поля

Поле Обязательно? Тип Описание
name Да string Имя скилла для вызова через /имя. Без пробелов, дефисы.
description Да string Краткое описание. Показывается в автодополнении.
model Нет string Какую модель использовать. По умолчанию — основная модель сессии.
tools Нет list Какие инструменты доступны скиллу. По умолчанию — все.

Поле model

Если скилл не требует мощной модели — можно указать более дешёвую:

model: claude-sonnet-4-20250514 # быстрая модель

Или наоборот — для сложных задач:

model: claude-opus-4-20250514 # мощная модель

Поле tools

Можно ограничить доступные инструменты. Например, для скилла, который только анализирует — разрешить только чтение:

tools: - Read - Glob - Grep

📝 Инструкции: тело скилла

После второго --- — это промпт для Claude. Пиши как обычную инструкцию: конкретно, с примерами, с ограничениями.

Что писать в инструкциях

  • Что делать: «Прочитай файл X, проанализируй Y, напиши Z»
  • Как делать: «Используй формат RFC, добавь type hints»
  • Чего не делать: «Не трогай файлы в папке migrations»
  • Формат результата: «Выведи список из 5 пунктов»
  • Примеры: Покажи идеальный результат

👧 Пример: простой скилл

Самый простой скилл — «приветствие с информацией о проекте»:

--- name: project-info description: Показывает информацию о проекте --- Прочитай файлы CLAUDE.md и package.json (или requirements.txt, Cargo.toml, go.mod) и кратко опиши проект: - Название и описание - Технологический стек - Как запускать - Как тестировать Выведи информацию в виде короткой сводки.

Сохрани как ~/.claude/skills/project-info/SKILL.md и вызывай:

/project-info

🛠️ Пример: полезный скилл

Скилл для генерации типизированных API-эндпоинтов:

--- name: gen-api-endpoint description: Генерирует REST API эндпоинт с тестами model: claude-sonnet-4-20250514 tools: - Read - Write - Edit - Bash --- Генерируй REST API эндпоинт по следующим правилам: ## Формат входных данных Пользователь опишет: метод (GET/POST/PUT/DELETE), путь, тело запроса, ответ. ## Что генерировать 1. **Маршрут** в формате FastAPI: - Pydantic-модель для request body (если есть) - Pydantic-модель для response (если есть) - Функция-обработчик с type hints - Docstring с описанием 2. **Тест** в файле `tests/test_{endpoint_name}.py`: - Тест на успешный ответ (200/201) - Тест на ошибку (404/422) - Использование pytest + httpx AsyncClient ## Ограничения - Не трогай существующие файлы маршрутов - Все модели клади в `schemas/` - Все маршруты в `routes/` - Форматирование: Black, линия до 88 символов ## Пример результата Если просят `POST /api/users` с `{name: str, email: str}`: - Создай `schemas/user.py` с UserCreate и UserResponse - Создай `routes/user.py` с create_user - Создай `tests/test_user.py` с тестами

Сохрани как ~/.claude/skills/gen-api-endpoint/SKILL.md и вызывай:

/gen-api-endpoint POST /api/orders с {product_id: int, quantity: int}

👆 Как вызывать

  1. Сохрани файл SKILL.md в папку: ~/.claude/skills/имя-скилла/SKILL.md
  2. Перезапусти Claude Code (или начни новую сессию)
  3. Набери /имя-скилла в чате

Структура папок

~/.claude/skills/ my-skill/ SKILL.md another-skill/ SKILL.md
💡 Подсказка: Набери / в чате и увидишь все доступные скиллы с описаниями. Кастомные будут помечены как «custom» (не «built-in»).

💡 Советы

Начни с простого

Первый скилл — что-то тривиальное: «покажи инфо о проекте» или «пробежись по коду и найди TODO». Потом усложняй.

Пиши инструкции как для стажёра

Конкретно, без двусмысленностей. Claude — умная модель, но ей нужно понимать контекст. Не «сделай хорошо», а «добавь type hints ко всем функциям в папке src/».

Тестируй итеративно

Написал скилл — попробуй вызвать. Видишь что Claude делает не то? Правь инструкции и вызывай снова. Это цикл: написал → проверил → поправил.

Разделяй скиллы по назначению

Один скилл — одна задача. Не пиши «сделай всё» — лучше три отдельных скилла: «генерируй эндпоинты», «генерируй тесты», «генерируй документацию».

Итого: Кастомный скилл — это файл SKILL.md с frontmatter (name, description, tools, model) и инструкциями. Кладёшь в ~/.claude/skills/, вызываешь через /имя.

Разница между скиллами и слэш-командами — в статье Skill vs Slash Command.
