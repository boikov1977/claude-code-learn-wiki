📋 Что такое Plan Mode?

Представь стройку. Есть архитектор — он рисует чертежи, считает нагрузки, проектирует фундамент. Но кирпичи он не кладёт и бетон не мешает. Это его работа — спроектировать, а строить будут другие.

Plan Mode — это когда Claude Code превращается в такого архитектора. Он только читает и анализирует код: смотрит файлы, ищет связи, изучает структуру. Но писать, редактировать или запускать команды — не может.

Проще простого: Plan Mode — это режим, в котором Claude Code изучает задачу и предлагает тебе план действий, ничего не меняя в коде. Как менеджер проекта, который собирает всё командой перед тем, как копать.

🎯 Зачем он нужен

Знаешь, почему многие разработчики любят Plan Mode? Потому что он решает сразу несколько проблем:

🔬 Сложная задача

Когда задача — не «поменяй кнопку», а «добавь авторизацию с ролями» — нужно сначала понять код, а потом уже писать. Plan Mode даёт время на изучение.

🗺️ Незнакомый проект

Попал в чужой код — не лезь сразу с пилой. Plan Mode поможет разобраться как всё устроено, какие зависимости, где критичные места.

⚡ Опасные изменения

Продакшн-код, миграции БД, тонкая логика — тут ошибка стоит дорого. Plan Mode показывает куда приведут изменения до того как они произойдут.

🤝 Договориться заранее

Когда не уверен в подходе — покажи план. Согласуй подход, обсуди альтернативы, укажи на риски. Потом уже — кодить.

Пошагово:

  1. Вход. Ты включаешь Plan Mode — Claude переключается в «только чтение».
  2. Исследование. Claude изучает код: читает файлы, ищет по grep, смотрит структуру папок.
  3. План. Claude составляет план: какие файлы трогать, что в них менять, какие есть риски.
  4. Показ. Ты видишь план на экране — читable и понятный.
  5. Утверждение. Ты правишь (если нужно) и утверждаешь.
  6. Реализация. Claude выходит из Plan Mode и начинает кодить по плану.

🚪 Как войти и выйти

Есть несколько способов — от самого быстрого до самого «настоящего»:

1. Команда в чате

Просто напиши /plan в поле ввода. Claude переключится в режим планирования.

Чтобы выйти и показать план на утверждение — используй ExitPlanMode.

2. Shift+Tab

Нажал Shift + Tab — режим переключился циклически. Удобно прямо во время работы, не отрываясь от задачи.

3. Флаг при запуске

Можно стартовать сразу в Plan Mode:

# Запуск сразу в режиме планирования claude --permission-mode plan
Действие Как
Войти в Plan Mode /plan в чате, Shift+Tab, или флаг --permission-mode plan
Выйти из Plan Mode ExitPlanMode — покажет план и переключит обратно
Переключить циклически Shift + Tab

🚫 Ограничения

Plan Mode — это не «бесплатный обед». Он что-то даёт, но и что-то забирает:

Что нельзя делать

Действие Доступно в Plan Mode?
Читать файлы ✅ Да
Искать по коду (grep, glob) ✅ Да
Веб-поиск (если разрешён) ✅ Да
Редактировать файлы 🚫 Нет
Создавать файлы 🚫 Нет
Запускать Bash-команды 🚫 Нет
Удалять файлы 🚫 Нет
⚠️ «А если мне нужен быстрый фикс?» — тогда Plan Mode бесит. Если ты знаешь что и где менять — просто правь код напрямую. Plan Mode хорош когда надо подумать, а не когда надо побыстрее.

Ситуация Использовать Plan Mode? Почему
Новая фича — «добавь авторизацию» Да Сложная задача, нужно спроектировать
Рефакторинг — «перепиши модуль» Да Много связей, нужно понять что затронется

📐 Что внутри плана

Когда Claude показывает план, он обычно выглядит так:

Пример типичного плана

# План: Добавить страницу профиля пользователя ## Шаг 1: Создать компонент UserProfile - Файл: src/components/UserProfile.tsx - Импорт существующего UserContext для данных - Типы: использовать interface User из src/types/user.ts ## Шаг 2: Добавить роутинг - Файл: src/App.tsx — добавить /profile маршрут - Защита маршрута через AuthGate ## Шаг 3: Стилизация - Файл: src/styles/profile.css — создать ## Риски - UserContext может не содержать все нужные поля — нужно проверить - Роутер уже использует lazy loading — новый компонент тоже стоит загружать лениво ## Время - Примерно 15-20 минут

Хороший план от Claude всегда содержит:

  • Конкретные файлы — не «где-то в коде», а точные пути
  • Что именно менять — импорты, функции, компоненты
  • Зависимости — что уже есть и что нужно учесть
  • Риски — что может пойти не так
  • Оценку времени — грубую, но полезную
💡 Совет: Если план показался слишком абстрактным или неполным — скажи Claude: «Добавь больше деталей по шагу 2» или «Проверь что X не сломается». Plan Mode как раз для этого и существует — чтобы править план до написания кода.

Итого: Plan Mode — это «сначала подумай, потом делай». Используй для сложных задач, незнакомого кода и опасных изменений. Для быстрых фиксов — не трать время, правь сразу. Plan Mode не заменяет мозг — он даёт время его включить. 🧠
