⏰ Что такое /loop?

Представь будильник. Поставил — он звонит каждые N минут, ты встаёшь и проверяешь что-то. Не надо каждый раз заново будильник ставить — он сам повторяется.

/loop — это команда, которая запускает задачу на повторение. Claude Code выполняет её с заданным интервалом иотчёт (отчитывается) о результатах — пока ты не скажешь остановиться.

Проще простого: /loop — это как сказать Claude: «Делай X каждые N минут и говори мне результат». Работает пока сессия жива. Быстро, просто, без лишнего кода.

Пошагово:

  1. Ввод. Пишешь /loop 5m проверь статус деплоя.
  2. Парсинг. Claude понимает: интервал — 5 минут, задача — проверить деплой.
  3. Ожидание. Таймер тикает. Claude молчит.
  4. Выполнение. Таймер сработал — Claude проверяет деплой.
  5. Отчёт. Claude пишет тебе результат: «Деплой прошёл, версия 2.4.1 на проде».
  6. Повтор. Ставим таймер заново и ждём следующего круга.
💡 Важно: Claude не прерывает текущий ответ ради loop. Он проверяет «не пора ли?» только когда ты пишешь следующее сообщение. Поэтому если ты молчишь час — задача не выполнится, пока не напишешь что-нибудь.

📝 Форматы команд

Есть четыре варианта использования — от самого гибкого до самого простого:

Команда Что делает Интервал
/loop 5m проверь статус деплоя Каждые 5 минут проверяет деплой Ты задаёшь
/loop проверь CI Claude сам выбирает интервал (обычно ~15 минут) Claude решает
/loop Дефолтный loop — доделать и почистить за собой Claude решает
напомни через час запустить миграцию Одноразовое напоминание (не повторяется) Один раз

Примеры из жизни

  • /loop 10m проверь что тесты проходят — мониторинг CI
  • /loop 2h напомни сделать код-ревью — периодическое напоминание
  • /loop 30s проверь не упал ли сервер — быстрый health-check
  • напомни через 30 минут сделать коммит — одноразовый таймер

🚫 Ограничения

/loop — это не «навсегда и бесплатно». Есть правила, которые надо знать:

Что ограничено

Ограничение Значение
Время жизни 7 дней — потом loop самоуничтожается
Сессия Работает только пока сессия открыта
Терминал Закрыл терминал — loop умер
Прерывание Не может проснуться посреди твоего ответа — только между твоими сообщениями
Остановка Напиши /stop или просто закрой сессию

Чего loop не может

  • Работать без сессии. Если Claude не запущен — нет loop. Это не cron.
  • Прерывать ответ. Если Claude только что тебе ответил и ты набираешь следующее — loop подождёт.
  • Запускаться сам. Loop не создаёт новых сессий. Он живёт только внутри текущей.
  • Превышать 7 дней. Сработает самоликвидация — дольше не дадут.
💡 Лайфхак: Если тебе нужен мониторинг, который работает 24/7 — loop не подойдёт. Для этого есть GitHub Actions, системный cron + claude -p, или другие механизмы. Loop — для текущей сессии.

📄 Кастомный loop.md

Дефолтный /loop (без аргументов) делает что-то общее: доделать незаконченное, почистить за собой. Но это можно перенастроить.

Как настроить

Создай файл ~/.claude/loop.md и напиши туда инструкции:

# ~/.claude/loop.md Каждые 15 минут: 1. Проверь статус ветки main — нет ли конфликтов 2. Посмотри не упал ли линтер 3. Если есть TODO-комментарии — напомни о них Одноразово (перед завершением): - Сделай git status и напомни про незакоммиченные изменения

Теперь когда ты просто пишешь /loop без аргументов — Claude выполняет то, что написано в loop.md, а не дефолтное поведение.

Вариант Что делает
/loop без loop.md Дефолт: доделать, почистить, привести в порядок
/loop с ~/.claude/loop.md То, что ты написал в loop.md
/loop 5m что-то конкретное Всегда выполняет твою команду, игнорируя loop.md

🔄 Альтернативы

Если loop не подходит по каким-то причинам — есть другие варианты:

GitHub Actions

Запуск по расписанию прямо в репозитории. Работает 24/7, не зависит от твоей сессии. Идеально для CI/CD мониторинга.

Системный cron + claude -p

Классический crontab + claude -p "задача". Каждые N минут запускается новая сессия. Надёжно, но дорого по API.

Уведомления в Slack/Telegram

Комбинируем cron + скрипт + webhook. Проверяем что-то и кидаем уведомление в мессенджер. Не нужен открытый терминал.

Интеграция с мониторингом

Prometheus + Grafana, Datadog, Uptime Robot — если речь про инфру. Claude тут не нужен, есть проверенные инструменты.

🏠 Что работает у нас

🚫

/loop через OpenCode НЕ работает

Мы выяснили: /loop у нас не функционирует. Причина — OpenCode-прокси не реализует инструменты CronCreate, CronList, CronDelete. А именно на них завязан весь механизм loop.

Когда ты пишешь /loop — ничего не происходит. Команда не доходит до Claude, потому что прокси не знает что с ней делать.

Что работает вместо этого

  • Системный cron + claude -p — надёжная замена. Настрой crontab -e и запускай Claude в non-interactive режиме.
  • GitHub Actions — если нужен мониторинг в облаке.
  • Просто напомни мне — скажи «напомни через час» и я попробую (если сессия будет жива).
# Пример: cron + claude для мониторинга деплоя # crontab -e */5 * * * * claude -p "Проверь статус деплоя на staging" >> ~/deploy-check.log

Итого: /loop — удобная штука для мониторинга в рамках сессии. Но у нас через OpenCode не работает. Для 24/7 мониторинга используй cron + claude -p или GitHub Actions. А loop.md на всякий случай создай — пригодится когда починят. ⏰
