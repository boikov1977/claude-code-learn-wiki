🔑 Что такое режим работы?

Представь: ты нанял помощника. Можно дать ему ключи от квартиры и сказать «делай что нужно». А можно сидеть рядом и разрешать каждое действие: «открой ящик — OK, включи дрель — подожди».

Режим работы (permission mode) — это именно эта настройка: насколько Claude Code может действовать самостоятельно, не спрашивая у тебя подтверждения.

Проще простого: Режим определяет, какие действия Claude Code делает молча, а какие — только после твоего «ОК». От этого зависит и скорость работы, и безопасность.

🎛️ Все пять режимов

⚪

default Стандартный

Базовый режим Claude Code. Спрашивает почти на каждое действие — чтение файлов, запуск команд, редактирование. Максимально безопасно, но медленно.

Для кого: Новички, первое знакомство с Claude Code, работа с незнакомым кодом.

🟢

auto Автоматический

Сам решает, что безопасно. Без операции (чтение, grep, навигация по коду) делает молча. Потенциально опасные (редактирование файлов, удаление, запуск команд) — спрашивает.

Для кого: Повседневная работа. Хороший баланс между скоростью и контролем.

Мы были на нём долго — и это нормальный выбор для большинства задач.

🟡

acceptEdits Авто-правки

Как auto, но с одним важным отличием: редактирование файлов разрешено без подтверждения. Чтение и запуск команд всё ещё могут спросить.

Для кого: Когда правишь много файлов подряд и не хочешь на каждый «Edit? Yes» отвечать. Экономит кучу кликов при массовых изменениях.

🟠

plan Режим планирования

Только читает. Не пишет, не редактирует, не запускает команды. Идеален для исследования кода и составления плана перед изменениями.

Для кого: Сложные задачи, где сначала нужно разобраться и спроектировать решение, а потом уже делать. Защита от случайных правок.

🔴

bypassPermissions Полный доступ

Делает всё без единого вопроса. Читает, пишет, удаляет, запускает — ничего не спрашивает. Самый опасный режим, но и самый быстрый.

Для кого: Опытные разработчики, которые знают, что делают. Когда ты уверен (уверен) в задаче и хочешь максимальной скорости.

⚠️ bypassPermissions ≠ «ничего не проверяется». Claude Code всё равно следует инструкциям из CLAUDE.md и системного промпта. Это не режим «самоуничтожения» — это режим «я доверяю агенту принимать решения самому».

🔄 Как переключаться

Переключить режим можно тремя способами — от самого быстрого до самого «настоящего»:

1. Shift+Tab в сессии (самый быстрый)

Нажал Shift + Tab — режим переключился циклически. Удобно прямо во время работы, не отрываясь от задачи.

2. Команда /режим в чате

Набери /mode и выбери нужный из списка. Более наглядно, чем Tab.

3. Настройка по умолчанию

Чтобы режим менялся по старту, настрой его в settings.json:

// ~/.claude/settings.json { "defaultMode": "bypassPermissions" }

4. Флаг при запуске

Можно передать режим прямо в командной строке:

# Запуск с автоматическим режимом claude --permission-mode auto # Запуск с полным доступом claude --permission-mode bypassPermissions # Запуск в режиме планирования claude --permission-mode plan

🏠 Что стоит у нас

🔴

Мы на bypassPermissions

Да, полный доступ. Причина простая: я всё равно каждый раз жму «да-да-да, согласен». Claude Code спрашивает, я подтверждаю — мы теряем время впустую.

Это записано в нашем CLAUDE.md и работает на практике уже давно. Если что-то пойдёт не так — всегда можно переключить через Shift+Tab.

🛡️ «А это безопасно?» — спросишь ты. Относительно. Мы компенсируем это другими способами: бэкапы JSON/YAML при каждом изменении (через хук), CLAUDE.md с чёткими инструкциями, и тем, что я знаю, что делаю. Не каждый разработчик может так — и это нормально.

Итого: Режим пермишенов — это не «навсегда». Меняй его под задачу: plan для анализа, auto для рутины, bypass для скорости. Главное — знать, что ты делаешь. 🎯
