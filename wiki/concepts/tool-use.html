🔍 Что такое Tool Use?

Claude Code — это не просто чат-бот, который отвечает текстом. Он умеет делать: читать файлы, писать код, запускать команды, искать по проекту. Всё это — инструменты (tools). Каждый раз, когда Claude хочет что-то сделать, он вызывает инструмент.

Простыми словами: Представь, что Claude — это программист за рабочим столом. У него есть отвёртки, ключи, линейки. Когда он хочет прочитать файл — берёт «лупу», когда хочет написать — берёт «ручку». Tool Use — это механизм, с помощью Claude берёт нужный инструмент и делает дело.

Это принципиально отличает Claude Code от обычного LLM-чата. Там ты просишь — он отвечает текстом. Здесь ты просишь — он выполняет, используя инструменты, и возвращает результат.

Пошагово:

  1. Список инструментов. При начале сессии Claude Code отправляет модели список доступных инструментов — каждый со своим описанием, именем и набором параметров.
  2. Решение. Модель анализирует задачу и выбирает подходящий инструмент. Например, для поиска файла — Glob, для чтения — Read.
  3. JSON-вызов. Модель формирует вызов в формате JSON: имя инструмента + параметры. Это не текст, а структурированные данные.
  4. Выполнение. Claude Code (клиент) получает JSON, находит нужный инструмент и выполняет действие.
  5. Результат. Результат (содержимое файла, вывод команды и т.д.) возвращается модели как следующее сообщение.
  6. Цикл. Модель решает: достаточно ли информации или нужно вызвать ещё один инструмент.
💡 Ключевой момент: Модель не видит файлов напрямую. Она получает только список инструментов и результаты их вызова. Вся «магия» — в правильном выборе инструмента и интерпретации результатов.

🔗 Цепочки вызовов (Tool Chains)

Одно действие почти всегда запускает цепочку. Claude не просто вызывает один инструмент и останавливается — он комбинирует их, как мастер комбинирует инструменты.

Типичные цепочки:

Цель Цепочка Пояснение
Найти и исправить баг Glob → Grep → Read → Edit → Bash Найти файлы → найти строку → прочитать контекст → исправить → проверить
Написать документацию Read → Write → Bash Прочитать код → написать README → проверить форматирование
Исследовать архитектуру → Glob → Grep → Read (x N) Посмотреть структуру → найти файлы → искать по содержимому → читать
Настроить MCP-сервер Read → Write → Bash → MCP-tool Прочитать конфиг → записать настройки → перезапустить → проверить

Claude сам решает, когда остановиться

Цепочка не задана жёстко. Модель после каждого шага оценивает: «Получил ли я достаточно информации? Достаточно ли я сделал?» Если да — отвечает пользователю. Если нет — вызывает следующий инструмент. Именно поэтому Claude Code может работать «автономно» 5-10 минут, выполняя десятки вызовов за один запрос.

⚠️ Осторожно с длинными цепочками: Чем длиннее цепочка, тем больше расходует токенов в context window. Если Claude зациклился — напиши /compact или останови через Esc.

Built-in инструменты

Встроены в Claude Code и доступны всегда (если не отключены):

Инструмент Что делает Тип
Read Читает файл (или его часть) Read-only
Write Создаёт или перезаписывает файл Write
Edit Частичное изменение файла (замена строки) Write
Bash Запускает shell-команду Exec
Glob Поиск файлов по шаблону Read-only
Grep Поиск по содержимому файлов (regex) Read-only
Agent Запуск субагента Agent
WebSearch Поиск в интернете Read-only
WebFetch Загрузка и парсинг URL Read-only
ListFiles Список файлов в директории Read-only

MCP-инструменты

Подключаются через MCP-серверы (Model Context Protocol). Это те инструменты, которые появляются, когда ты добавляешь внешние серверы через claude mcp add или настраиваешь в .mcp.json.

Пример: Наш poisk-mcp добавляет инструмент search_web — поиск в интернете. Без этого сервера этого инструмента просто нет в списке.

Skills (инструменты из скиллов)

Кастомные скиллы (в ~/.claude/skills/) тоже предоставляют инструменты. Например, OpenSpec-скиллы добавляют openspec-propose, openspec-apply и другие. Они вызываются через Agent или напрямую.

💡 Как посмотреть все доступные инструменты: Напиши /tools — Claude покажет built-in инструменты. /mcp — покажет MCP-инструменты.

⏳ Рейт-лимиты и ограничения

Tool Use — это не безлимитный карнавал. Есть границы:

Лимиты на вызовы

Ограничение Значение
Вызовов за один запрос ~60-80 (зависит от модели и провайдера)
Пауза между вызовами Есть delay/backoff — искусственная задержка
Контекстное окно Каждый вызов и результат расходуют токены. При заполнении — /compact
Таймаут Bash 120 секунд (по умолчанию), максимум 600 секунд
Размер файла (Write) Нет жёсткого лимита, но очень большие файлы расходуют много токенов

Что происходит при превышении

  • Рейт-лимит API: Ошибка 429 (Too Many Requests). Claude Code автоматически делает retry с exponential backoff.
  • Контекст заполнен: Claude начнёт «забывать» старые части диалога. Помогает /compact.
  • Слишком длинная цепочка: Модель может сама остановиться, решив что информации достаточно.
  • Таймаут Bash: Команда принудительно завершается.
💡 Лайфхак: Если Claude «застрял» и зациклился — нажми Esc (прервать текущий ответ) или напиши /compact (сжать контекст). Это не ломает ничего — просто останавливает бесконечный цикл.

🚫 Ограничения отдельных инструментов

Каждый инструмент имеет свои «хотелки» и ограничения:

Инструмент Ограничение Почему
Write Не может перезаписать файл без предварительного Read Защита от потери данных. Клиент проверяет, что файл был прочитан.
Edit Точное совпадение old_string Строка должна совпадать точно, включая пробелы и переносы.
Bash Не поддерживает интерактивные команды Нет TTY: htop, vim, git rebase -i не работают.
Bash Запрещённые команды (по умолчанию) rm -rf /, sudo и другие опасные команды блокируются.
Agent Субагент в изолированной песочнице Не видит весь контекст основной сессии. Каждый субагент начинает «с чистого листа».
Read Максимум 2000 строк за вызов Ограничение на размер. Для больших файлов нужен offset + limit.
MCP-инструменты Зависят от доступности сервера Если MCP-сервер упал — инструмент недоступен.
WebSearch Не работает через все провайдеры Например, DeepSeek выдаёт ошибку схемы (у нас решено через poisk-mcp).

allowedTools и deniedTools

В настройках сессии можно явно разрешить или запретить конкретные инструменты:

# Пример: разрешить только чтение и поиск # В .claude/settings.local.json или через CLI { "allowedTools": ["Read", "Glob", "Grep", "Bash(git log:*)"], "deniedTools": ["Write", "Edit"] }

Это полезно, когда хочешь чтобы Claude только анализировал, но не трогал код. Также можно ограничить Bash конкретными командами через паттерн: Bash(git log:*)

🎯 Режимы и инструменты

Режим работы Claude Code определяет, какие инструменты доступны:

📖 Plan Mode

Только read-only инструменты:

  • Read — чтение файлов
  • Glob — поиск файлов
  • Grep — поиск по содержимому
  • Bash — только безопасные команды

Цель: исследовать, не трогая код. Составить план перед работой.

⚙️ Auto Mode

Все разрешённые инструменты:

  • Read, Write, Edit, Bash
  • Glob, Grep
  • Agent, WebSearch
  • MCP-инструменты

Цель: полная работа. Claude читает, пишет, запускает, проверяет.

⚡ bypassPermissions

Все инструменты без подтверждения:

  • Никаких промптов «Подтвердить?»
  • Claude действует сразу
  • Риск: может случайно удалить файл

Цель: максимальная скорость. Для опытных пользователей.

🔒 Ручное подтверждение

Стандартный режим:

  • Read, Glob, Grep — автоматически
  • Write, Edit, Bash — с подтверждением
  • Ты контролируешь каждый шаг

Цель: баланс скорости и контроля. Рекомендуется по умолчанию.

Итого: Tool Use — это ядро Claude Code. Без инструментов это просто LLM с контекстом. С инструментами — это рабочий, который читает, пишет, запускает и проверяет. Знай ограничения, используй режимы с умом, и Claude сделает за тебя чёрную работу.
