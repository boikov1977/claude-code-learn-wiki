📌 Что такое Frontmatter?

Представь: ты создаёшь нового сотрудника. Ему нужно дать имя, объяснить чем занимается, дать инструменты и ограничить, чего делать нельзя. Именно это и делает frontmatter — YAML-шапка в самом начале .md-файла агента.

Проще простого: Frontmatter — это блок между двумя тройными дефисами (---) в начале Markdown-файла. Claude Code читает его первым делом и понимает, с кем имеет дело.

--- name: my-agent description: Делает полезные штуки model: inherit --- # Вот тут начинается системный промпт агента Ты — агент, который...

Когда ты вызываешь агента через /agents или Claude сам решает делегировать задачу, он сначала парсит frontmatter, а потом уже передаёт управление агенту с учётом его конфигурации.

💡 Если поле не указано — Claude Code берёт значение из сессии или дефолт. Единственное, что реально нужно для работающего агента — это системный промпт в теле файла. Всё остальное — опциональная настройка.

📋 Все поля по порядку

обязательное name

Уникальный идентификатор агента. Используется в вызовах и логах. Без пробелов, латиницей, через дефис.

name: writer-agent name: innovator name: explorer

обязательное description

Описание когда делегировать задачу этому агенту. Claude Code использует его для принятия решения: «Кому отдать эту задачу?». Пиши action-oriented.

description: > Быстрый поиск и анализ кода. Использовать когда нужно найти файлы, изучить структуру, понять как устроен модуль. Read-only — ничего не меняет.

опциональное tools

Список инструментов, которые агенту разрешено использовать. Если не указать — агент наследует все инструменты родительской сессии, включая MCP.

tools: - Read - Write - Edit - Bash - Glob - Grep

Важно: Skill и Agent тоже нужно перечислить явно, если агенту нужны скиллы или вызов других агентов!

опциональное disallowedTools

Что агенту категорически нельзя, даже если инструмент наследован. Идеально для read-only агентов.

# Explorer: только читать, писать нельзя disallowedTools: - Write - Edit - NotebookEdit - Skill - Agent

опциональное model

Какую LLM использовать. Три варианта:

  • inherit — наследовать модель из текущей сессии (экономия!)
  • Короткое имя: sonnet, opus, haiku
  • Полный ID: claude-go-mimo-v2.5, claude-go-deepseek-v4-pro

опциональное effort

Насколько глубоко агент должен «думать». Влияет на количество токенов и время ответа.

ЗначениеКогда использовать
lowБыстрые ответы, рутинные задачи
mediumСтандартная работа
highСложные задачи, нужен тщательный анализ
xhighМаксимальная глубина (writer-agent так работает!)

опциональное permissionMode

Как агент работает с пермишенами:

  • auto — делает всё без вопросов (если разрешено инструментами)
  • accept — каждый шаг на подтверждении
  • bypass — обходит все проверки (опасно!)
  • plan — только планирует, не выполняет

опциональное mcpServers

MCP-серверы, которые поднимаются только когда этот агент активен. Это изоляция! Сервер не жрёт ресурсы, пока агент не вызван.

mcpServers: - name: context7 type: remote url: https://mcp.context7.com/mcp headers: CONTEXT7_API_KEY: ctx7sk-... # Или локальный: - name: poisk-mcp command: python args: [server.py]

опциональное skills

Скиллы, которые предзагружаются агенту. Он сможет вызывать их через инструмент Skill.

skills: - openspec-propose - openspec-explore - deep-research

остальные hooks, maxTurns, memory, isolation, color

ПолеТипОписание
hooksobjectХуки, специфичные для агента (PreToolUse, PostToolUse и т.д.)
maxTurnsnumberЛимит шагов. Защита от зацикливания (например, maxTurns: 30)

Пример: Наш writer-agent имеет в frontmatter model: claude-go-mimo-v2.5 и effort: xhigh. Значит он всегда работает на mimo и с максимальным effort — даже если основная сессия на другой модели.

🎯 Хитрости и лайфхаки

1. description — самое важное поле

Claude Code решает, кому делегировать, именно по description. Пиши action-oriented:

  • ✅ «Использовать когда нужно найти файлы, изучить структуру, понять как устроен модуль»
  • ❌ «Агент для поиска» — слишком размыто

2. Если не указать tools — агент получает ВСЁ

Включая MCP-серверы, скиллы, Write, Bash — всё подряд. Это удобно, но опасно: агент может случайно удалить файлы. Лучше всегда явно указывать.

3. model: inherit — экономия на подписке

Если не нужно, чтобы агент работал наопределённыйной модели — ставь inherit. Он подхватит модель из текущей сессии. Не платишь за лишние API-вызовы на дорогой модели.

4. MCP-серверы изолированы по агенту

MCP-серверы из frontmatter поднимаются только когда этот агент активен. Это значит, что context7 у innovator не жрёт память, пока работает writer-agent. Серверы из глобального .claude.json подняты всегда.

⚡ maxTurns — твой друг. Без лимита агент может зациклиться и потратить кучу токенов. Для простых агентов 15–30 шагов достаточно. Для сложных workflows (openspec-manager) лучше не ставить лимит — он сам знает, когда остановиться.
🔒 disallowedTools надёжнее, чем просто не включать в tools. Если агент наследует все инструменты (не указан tools), то disallowedTools — единственный способ заблокировать Write/Edit. Как у explorer.

Итого: Frontmatter — это не магия, а просто конфиг. Заполнил шапку → получил агента. Не заполнил — получил дефолтного. Всё просто, но мощно. 🚀
