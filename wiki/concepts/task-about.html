Task Tool — todo-лист внутри Claude Code

Встроенный инструмент для отслеживания прогресса в рамках одной сессии

📋 Что такое Task

Task — это встроенный инструмент Claude Code, который работает как todo-лист прямо в коде. Не про под-агентов, не про Workflow tool — просто чек-лист для отслеживания прогресса внутри одной сессии.

Зачем нужен: когда задача многошаговая и легко потерять нить — что уже сделано, что в работе, что осталось. Task фиксирует это и показывает прямо в интерфейсе.

Полезен, когда я создаю задачи, отмечаю прогресс, а ты видишь что сделано и что в работе. Никакого кода, никаких запросов — только фиксация состояния.

🔧 Что умеет Task

Task предоставляет три операции:

TaskCreate

Создать задачу. Указываешь название, описание — статус автоматически ставится pending.

TaskCreate({ "name": "Настроить хуки", "description": "Добавить PreToolUse хуки для бэкапов" }) // Статус: pending

TaskUpdate

Обновить статус задачи или отредактировать её. Переключаешь между in_progress, completed, cancelled.

TaskUpdate({ "id": "task-1", "status": "in_progress" })

TaskList

Посмотреть все активные задачи. Полезно, когда нужно вспомнить контекст после перерыва.

TaskList() // Возвращает список всех задач с их статусами

👁 Где видно

Задачи отображаются снизу в интерфейсе — панель с активными задачами и их статусами. Можно также вызвать список вручную командой /tasks.

Визуально:

  • pending — ожидает выполнения
  • in_progress — в работе
  • completed — выполнено
  • cancelled — отменено

💡 Когда я использую Task

  • Многошаговая задача (3+ шагов): создать субагента, настроить хуки, проверить — чтобы не забыть промежуточные шаги
  • Можно потерять нить: "что я уже сделал, что осталось" — Task сохраняет контекст
  • Нужен прогресс в реальном времени: вижу, как задача переходит от pending к completed
Практический пример: "Установи и настрой X, Y, Z" — создаю три задачи, каждую отмечаю in_progress перед выполнением, после выполнения ставлю completed. В конце вижу полную картину.

⚠️ Отличие от Agent и Workflow

Важно понимать: Task — это не вычислительный ресурс, а простой чек-лист.

Инструмент Что это Контекст
Task Todo-список Текущая сессия
Agent Под-агент Отдельный контекст
Workflow Скрипт с несколькими агентами Много отдельных контекстов
Помните: Task не отправляет код и не делает запросы. Он только фиксирует мой прогресс. Это не замена Agent и не замена Workflow — это дополнение к ним.

✅ Best Practices

1
Создавай задачи перед началом большой работы

Не создавай задачи "на ходу" — сначала опиши все шаги, потом приступай.

2
Держи в in_progress только одну задачу за раз

Если две задачи "в работе" одновременно — теряется ясность. Сделай одну, потом следующую.

3
Помечай completed сразу после выполнения

Не откладывай "когда-нибудь" — пометка сделано сразу. Иначе задача "висит" и сбивает с толку.

4
Не плоди задачи

3-5 задач — норма. 15 — перебор. Если задач больше — разбей на подзадачи или используй Workflow.

📖 Пример использования

Сценарий: "Установи и настрой X, Y, Z"

Step 1: Создать 3 задачи TaskCreate({ name: "Установить X", description: "..." }) → pending TaskCreate({ name: "Настроить Y", description: "..." }) → pending TaskCreate({ name: "Проверить Z", description: "..." }) → pending Step 2: Первая задача → in_progress TaskUpdate({ id: "task-1", status: "in_progress" }) Step 3: Выполнить X ... выполняю задачу ... Step 4: completed → следующая in_progress TaskUpdate({ id: "task-1", status: "completed" }) TaskUpdate({ id: "task-2", status: "in_progress" }) Step 5: Повторяю для Y и Z

🚫 Чего Task НЕ делает

  • Не запускает код или команды — это только список задач, не исполнитель
  • Не вызывает под-агентов — для этого есть Agent tool
  • Не делает ничего сам — только фиксирует мой прогресс

Task — это чек-лист, не вычислительный ресурс. Он не отправляет код и не делает запросы.

Task — это встроенный todo-лист в Claude Code. Он не заменяет Agent и Workflow, а дополняет их: пока Agent выполняет работу, Task отслеживает прогресс. Создавай задачи перед большой работой, держи по одной in_progress, помечай completed сразу. 3-5 задач — идеально.
