🧩 Что такое контекстное окно

Контекстное окно (Context Window) — это рабочая память Claude. Всё, что находится внутри — я вижу, помню и могу использовать. Всё, что оказалось за пределами — потеряно (пока ты не скажешь мне заново).

Проще простого: Представь письменный стол определённого размера. На нём умещается только определённое количество бумаг. Положил новую — старую придётся убрать. Контекст — это этот стол. У нас ~200K токенов, зависит от модели.

«Токен» — это примерноодна треть слова. На 200K токенов помещается примерно 500 страниц текста. Звучит много, но когда ты читаешь десятки файлов, запускаешь команды и ведёшь длинный диалог — стол заполняется быстрее, чем кажется.

💡 Ключевое: Контекст не бесконечный. Это ресурс, которым нужно управлять. Осознанное управление контекстом — один из главных навыков работы с Claude Code.

⚠️ Biggest Consumer — история диалога

История диалога — это то, что растёт быстрее всего. Каждое сообщение, каждый ответ, каждый вызов инструмента с результатом — всё накапливается. Именно поэтому длинные сессии «давят» контекст.

⚙️ Команды управления контекстом

Claude Code даёт несколько команд для управления контекстом. Вот все:

/compact

Сжатие истории. Claude пересказывает самое важное из прошлого и выбрасывает детали.

  • Экономит место
  • Теряются нюансы (код, логи, конкретика)
  • Не стирает CLAUDE.md, память или системные инструкции
  • Можно указать что запомнить: /compact запомни что мы выбрали PostgreSQL

/clear

Полная очистка. Начинаем с чистого листа. Ничего не помню.

  • Контекст = только системный промпт
  • Идеально когда задача завершена
  • Нет способа вернуть удалённое

/rewind

Откат на N шагов назад. Отменяет последние действия, как undo.

  • Возвращает контекст к состоянию N ходов назад
  • Полезно когда что-то пошло не так

--resume / -r

Продолжить прерванную сессию. Claude Code сохраняет историю на диск.

  • Загружает предыдущий контекст
  • Можно вернуться к сессии через несколько дней

--continue / -c

Продолжить самую свежую сессию. Быстрый способ вернуться — без выбора из списка.

  • Находит последнюю сессию в текущей папке
  • Загружает её контекст
  • Работает как --resume, только автоматически
# Всё про команды управления контекстом # Сжать историю (самая частая команда) /compact # Сжать, но сохранить важное /compact запомни что мы обсудили архитектуру очереди # Полная очистка — новая сессия /clear # Откат на 3 шага назад /rewind 3 # Продолжить конкретную сессию по ID claude --resume abc123 # Продолжить последнюю сессию в этой папке claude --continue

✅ Компактить стоит когда:

  • Процент заполнения > 80-90% — я сам скажу «контекст заполнен»
  • Чувствуешь «забывчивость» — я стал повторяться или забывать то, что обсуждали в начале
  • Планируешь долгий разговор — хочешь освободить место под текущую задачу

❌ Не надо компактить когда:

  • Сессия короткая и всё помещается
  • Важна каждая деталь — лучше разбей на несколько сессий
  • Можно использовать субагента — вынеси детали в его контекст

💾 Как не терять важное

Главный страх при компакте — потерять что-то ценное. Вот три способа обезопаситься:

1. Скажи «запомни» перед /compact

Самый простой способ. Прямо перед компактом напиши: /compact запомни что мы выбрали PostgreSQL и API будет REST. Я включу это в краткое резюме.

2. Попроси записать в файл

«Запиши вывод анализа в docs/analysis.md» — и файл никуда не денется. Даже после /clear он останется на диске.

3. Используй CLAUDE.md / память

Если что-то критически важное — попроси записать в MEMORY.md или CLAUDE.md. Это загружается при каждом старте сессии.

Главный принцип: Если информация важна — пусть она будет не в контексте, а на диске. Файл никуда не денется. Контекст — временное хранилище.

🤖 Субагенты как решение проблемы контекста

Каждый субагент имеет свой собственный контекст. Вся его «грязь» — логи, поиск, промежуточные чтения файлов — остаётся в нём. Я получаю только финальный результат.

Почему это круто

  • Субагент может прочитать 50 файлов, прогонять grep и find — а я увижу только ответ
  • Его контекст «грязный» — но мой остаётся чистым
  • Это главный инструмент управления контекстом в Claude Code
# Пример: субагент исследует код, а ты получаешь чистый ответ # Без субагента (засоряет контекст): > Прочитай src/auth.py, src/models.py, src/api.py > Сравни подходы аутентификации # = 3 файла × ~500 строк = ~1500 строк в контексте # С субагентом (контекст чистый): > Запусти explorer с запросом: «сравни подходы аутентификации в src/» # = 1-2 абзаца ответа в контексте

🎯 Практические советы

1. Большие файлы — читай по частям

Файл на 2000 строк? Используй offset/limit, чтобы читать кусками. Прочитал первые 200 — этого достаточно? Не надо тащить остальные 1800 в контекст.

2. Разбивай большие задачи на шаги

Вместо «перепиши весь проект» — «обнови модуль auth». Каждый шаг — отдельная фаза. Между ними можно компактить или начать новую сессию.

3. Используй субагентов для исследований

«Найди все места где используется X» — это задача для explorer, а не для основного контекста. Субагент потратит свой контекст на поиск, а ты получишь только список.

4. Сохраняй выводы перед компактом

Если ты получил ценную информацию из длинного процесса (анализ, тесты, сравнение) — попроси записать её в файл до компакта. Потом обратно не вернёшь.

5. Новая тема — новая сессия

Закончил с бэкендом и переходишь на фронтенд? Закрой сессию и открой новую. Чистый контекст без «мусора» от предыдущей задачи.

6. Следи за индикатором контекста

Claude Code показывает процент заполнения контекста. Если видишь что приближается к 90% — пора действовать: компактить, сохранять, начинать новую сессию.

Итого: Контекст — это рабочая память Claude. Видит только то, что внутри. /compact сжимает, /clear очищает, субагенты выносят грязь в свой контекст. Важное — сохраняй на диск. Файл не денется, а контекст — временный. 🧩
