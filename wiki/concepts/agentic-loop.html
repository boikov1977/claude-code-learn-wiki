💡 Что такое агентный цикл

Когда ты отправляешь промпт Claude Code — он не просто «отвечает». Он запускает цикл, в котором последовательно выполняет шаги: читает файлы, анализирует код, пишет изменения, запускает проверки и повторяет, пока задача не будет решена.

Простыми словами: Агентный цикл — это рабочий процесс «прочитал → подумал → сделал → проверил → повторил». Одна задача может занимать от 5 до 50 итераций этого цикла. Ты видишь результат в конце — Claude работает автономно, а ты контролируешь ключевые решения.

Это то, что отличает Claude Code от обычного чата: в чате один вопрос — один ответ. В агентном цикле один вопрос — серия действий, которая заканчивается готовым результатом.

🎯 Фазы цикла

Каждая итерация агентного цикла проходит через несколько фаз:

1

Приём промпта

Пользователь отправляет задачу. Claude анализирует её и определяет стратегию: какие файлы читать, какие команды запускать, что писать.

2

Исследование (чтение)

Claude читает связанные файлы через инструменты Read, Glob, Grep. Понимает структуру проекта, находит зависимости, определяет контекст.

3

Анализ и план

На основе прочитанного Claude формулирует план действий: что именно нужно изменить, в каких файлах, какими инструментами.

4

Выполнение (код)

Claude применяет изменения: Edit для правок, Write для новых файлов, Bash для команд. Может комбинировать несколько инструментов в одной итерации.

5

Проверка

Claude проверяет результат: запускает тесты, проверяет форматирование, читает логи ошибок. Если что-то не так — возвращается к шагу 2.

6

Решение о повторе

Claude решает: достаточно ли информации? Задача решена? Если да — отвечает пользователю. Если нет — повторяет цикл с шага 2.

Типичная длина: Простая задача (написать функцию) — 5–10 итераций. Средняя (исправить баг с тестами) — 10–25. Сложная (добавить фичу с рефакторингом) — 25–50 итераций.

🧩 Компоненты цикла

Три ключевых механизма, из которых состоит агентный цикл:

Tool Use — инструменты

Tool Use — это механизм, с помощью которого Claude вызывает внешние инструменты. Каждое действие (прочитать файл, запустить команду, записать код) — это вызов инструмента. Claude сам выбирает нужный инструмент и формирует вызов в формате JSON.

Основные инструменты:

  • Read — чтение файла
  • Write — создание / перезапись файла
  • Edit — точечное изменение файла
  • Bash — запуск shell-команды
  • Glob / Grep — поиск файлов и содержимого
  • Agent — вызов субагента

Подробнее — в статье Tool Use.

Context Window — рабочая память

Context Window — это «рабочая память» Claude. Всё, что он прочитал, написал и получил от системы — помещается в контекстное окно. Каждый вызов инструмента и его результат расходуют токены.

Когда контекст заполняется — Claude может начать «забывать» ранние шаги. В этом случае помогает /compact (сжатие контекста).

Подробнее — в статье Context Window.

Permissions — контроль доступа

Permissions — это система правил, которая определяет, какие действия Claude может выполнять автоматически, а какие требуют подтверждения. Read-only операции проходят свободно, Write/Edit/Bash — обычно с вопросом.

Ты можешь настроить правила (allow/deny/ask), чтобы Claude не спрашивал разрешения на routine-действия, но сохранял контроль над опасными.

Подробнее — в статье Permissions.

👩‍💻 Пример: пошаговый разбор

Задача: «Напиши функцию для подсчёта строк в файле и тест к ней». Вот как проходит агентный цикл:

# Итерация Действие Claude Инструмент
1 Исследование Claude смотрит структуру проекта: какие языки, фреймворки, система тестирования Bash (ls), Glob
2 Анализ Находит существующие утилиты и модули. Решает: создать отдельный модуль utils.py Grep, Read
3 Код: функция Пишет функцию count_lines(filepath) в utils.py Write
4 Код: тест Создаёт test_utils.py с тестом для функции Write
5 Проверка Запускает тесты: pytest test_utils.py -v Bash
6 Решение Тесты прошли. Claude отвечает: «Функция и тест готовы. Вот что я сделал: ...» —

Если бы тесты упали...

На шаге 5 Claude увидел бы ошибку. Он бы прочитал стектрейс (шаг 6), нашёл проблему (шаг 7), исправил код (шаг 8), запустил тесты снова (шаг 9) — и повторял бы до тех пор, пока тесты не пройдут. Ты бы увидел только финальный результат.

⚠️ Ограничения

Ограничение Значение Что делать
Максимум итераций Зависит от модели и провайдера (~60–80 tool-вызовов) Если Claude «застрял» — Esc для прерывания
Context window 128K–200K токенов (зависит от модели) /compact для сжатия контекста
Таймаут Bash 120 секунд (по умолчанию) Длинные команды: --timeout 600
Память между сессиями Нет: контекст не сохраняется между запусками Используй CLAUDE.md для важных инструкций

🏗️ Как управлять циклом

Дай контекст в промпте

Чем точнее промпт — тем меньше итераций. Вместо исправь баг — в файле utils.py функция parse_json падает с KeyError на строке 42 когда приходит пустой массив.

Используй Plan Mode для сложных задач

Для крупных фичей сначала войди в Plan Mode — Claude исследует код и составит план без изменения файлов. Потом одобри план и он начнёт выполнение.

Следи за контекстом

Если задача большая и контекст заполняется — вызови /compact или разбей задачу на подзадачи. Claude может «забыть» ранние шаги, если контекст переполнен.

Останавливай вовремя

Если Claude зациклился или делает лишнее — нажми Esc. Ты ничего не потеряешь: все изменения уже применены, просто итерации прекратятся.

Итого: Агентный цикл — это ядро Claude Code. Промпт → исследование → код → проверка → повтор. 5–50 итераций, всё автономно. Управляй через точные промпты, Plan Mode и /compact.
