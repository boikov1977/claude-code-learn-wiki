🔧 Что такое хуки?

Представь: ты настроил уведомления в телефоне -- «когда кто-то написал в Telegram, покажи пуш». Хуки в Claude Code работают точно так же. Только вместо пушей -- выполняется команда.

Проще простого: Хук -- это автоматическая команда, которая выполняется при определённом событии. Claude вызвал Write - сработал хук - сделал бэкап. Всё. Никаких агентов, никакого AI -- просто shell-команда на событии.

Хуки настраиваются в JSON-конфигах и работают на уровне Claude Code, не требуя запуска моделей. Это делает их быстрыми и надёжными.

🔄 Жизненный цикл хука

 flowchart LR
   A["Claude вызывает инструмент"] --> B{"Есть хук на событие?"}
   B -->|"Нет"| C["Инструмент выполняется"]
   B -->|"Да, matcher совпал"| D["Запуск action"]
   D --> E{"PreToolUse?"}
   E -->|"Да"| F{"Exit code = 0?"}
   F -->|"Да"| C
   F -->|"Нет"| G["Инструмент заблокирован"]
   E -->|"Нет, PostToolUse"| C
   C --> H["PostToolUse action"]
   H --> I["Готово"]
   G --> I

Процесс идёт по шагам:

  1. Событие. Claude вызывает Write, Edit, Bash -- что угодно.
  2. Проверка. Claude Code смотрит: есть ли хук на это событие и совпадает ли matcher?
  3. Выполнение. Если совпал -- запускается указанный action (команда, LLM-проверка и т.д.).
  4. Результат. Для PreToolUse: если команда вернула block или ненулевой код -- инструмент не вызывается. Для PostToolUse результат -- уже неважен (действие выполнено).
💡 Ключевой момент: PreToolUse может заблокировать вызов инструмента. PostToolUse -- уже нет, потому что действие уже произошло.

📋 Типы событий (триггеры)

Каждый хук привязан к событию. Claude Code поддерживает несколько типов:

Событие Когда срабатывает Особенности
PreToolUse Перед вызовом инструмента (Write, Edit, Bash...) Можно заблокировать вызов. Получает JSON с tool_input
PostToolUse После успешного выполнения инструмента Нельзя отменить. Получает tool_response
PostToolUseFailure После ошибки в инструменте Полезно для логирования ошибок
SessionStart При старте сессии Раз для раза. Инициализация
Stop При остановке (clear, exit) Можно использовать для cleanup
PrePrompt Перед отправкой промпта модели Можно модифицировать промпт
UserPromptSubmit Когда пользователь отправил сообщение Валидация ввода
Notification Когда Claude хочет уведомить пользователя Системные уведомления
PreCompact / PostCompact До/после сжатия контекста Управление контекстом
PermissionRequest При запросе разрешения Авто-ответ на типичные запросы

Matcher (фильтр) использует регулярные выражения. Например, Write|Edit ловит оба типа инструмента. Можно указать .* чтобы ловить все.

⚙ Что умеют делать хуки

command

Выполнить shell-команду. Самый распространённый и простой.

{ "type": "command", "command": "echo 'Файл изменён'" }

Команда получает JSON на stdin с данными о событии. Возвращает exit code: 0 = OK, не 0 = блокировка (для PreToolUse).

prompt

Проверить условие через LLM. Дорого, но умно.

{ "type": "prompt", "prompt": "Проверь, что код\nсоответствует стилю проекта" }

Модель анализирует контекст и принимает решение. Подходит для сложной валидации.

http

Отправить HTTP-запрос. Уведомления в Slack, Telegram и т.д.

{ "type": "http", "url": "https://hooks.slack.com/...", "method": "POST" }

Полезно для нотификаций команде, интеграции с CI/CD.

agent

Запустить субагента для проверки. Мощно, но медленно.

{ "type": "agent", "agent": "my-checker", "prompt": "Проверь PR" }

Субагент получает контекст и может принимать решения на уровне Claude.

mcp_tool

Вызвать инструмент из подключённого MCP-сервера. Позволяет использовать любые MCP-инструменты как хуки.

{ "type": "mcp_tool", "server": "my-server", "tool": "my_tool", "input": { "key": "value" } }

📁 Где живут и как настроить

Хуки настраиваются в JSON-файлах. Три уровня:

 flowchart TD
   subgraph Global["Глобальные настройки"]
     A["~/.claude/settings.local.json"]
   end
   subgraph Project["Проектные настройки"]
     B[".claude/settings.json"]
   end
   subgraph Agent["Агентные настройки"]
     C["~/.claude/agents/*.md"]
   end
   A --> D["Действуют во всех проектах"]
   B --> E["Действуют только в этом проекте"]
   C --> F["Действуют только когда работает агент"]
   D --> G["Хуки складываются из всех трёх уровней"]
   E --> G
   F --> G
Файл Область Когда
~/.claude/settings.local.json Глобальные Во всех проектах, для всех сессий
.claude/settings.json Проектные Только в этом проекте
Frontmatter агента (~/.claude/agents/*.md) Для агента Только когда работает этот агент

Формат записи хуков:

// ~/.claude/settings.local.json { "hooks": { "/* тип события */": [ { "matcher": "/* регулярка: Write|Edit, Bash, ... */", "hooks": [ { "type": "command", "command": "/* shell-команда */" } ] } ] } }
💡 Не путай: settings.local.json -- глобальные настройки (обычно gitignore). .claude/settings.json в проекте -- проектные. Хуки в обоих файлах складываются, не перезаписываются.

🛠 Наши хуки в действии

У нас настроены два хука в ~/.claude/settings.local.json. Оба реагируют на запись файлов:

Бэкап перед записью

PreToolUse на Write|Edit: копирует JSON/YAML в /tmp/claude-backups/ перед изменением.

Позволяет откатить изменения, если что-то пошло не так.

Автоформатирование

PostToolUse на Write|Edit: запускает prettier после записи JSON/YAML-файлов.

Код всегда единообразен, независимо от того как написал Claude.

Бэкап перед записью

Тип: PreToolUse

Matcher: Write|Edit

Что делает: Перед записью JSON/YAML-файла сохраняет текущую версию в /tmp/claude-backups/ с меткой времени.

# Структура бэкапов: /tmp/claude-backups/ +-- 1750087234-settings.json +-- 1750087300-config.yaml +-- 1750087456-package.json

Зачем: Если Claude перезапишет конфиг не так -- можно откатить.

Автоформатирование

Тип: PostToolUse

Matcher: Write|Edit

Что делает: После записи JSON/YAML-файла запускает npx prettier --write для форматирования.

# Что форматируется: *.json - красивые отступы *.yaml - единый стиль *.jsonc - комментарии сохраняются

Зачем: Claude пишет JSON корректно, но не всегда красиво. Prettier делает единообразно.

Общая черта: защита от падения

Оба хука завершаются || true. Это значит: даже если Python упал или Prettier не установлен -- хук не заблокирует запись файла. Claude продолжит работу.

# В конце каждой команды: ... 2>/dev/null || true # 2>/dev/null -- подавляем stderr # || true -- всегда возвращаем exit 0

💡 Зачем вообще нужны

Бэкапы и форматирование -- это два самых популярных сценария. Но хуки умеют больше:

Бэкапы

Копировать файл перед изменением. Если что-то пошло не так -- есть куда откатиться.

Пример: наш PreToolUse на Write|Edit.

Автоформатирование

Запускать linter/formatter после записи. Код всегда чистый, не зависимо от того что написал Claude.

Пример: наш PostToolUse на Write|Edit + Prettier.

Логирование

Записывать все Bash-команды в лист-журнал. Удобно для аудита: что делал Claude, когда, и с какими аргументами.

// PreToolUse matcher="Bash" "command": "echo \"$(date -Iseconds) $(cat | jq -r '.tool_input.command')\" >> ~/claude-audit.log"

Уведомления

Дёрнуть Slack/Telegram когда Claude завершил задачу. Удобно когда ждёшь результат.

// Stop-хук "type": "http", "url": "https://hooks.slack.com/services/...", "method": "POST", "body": "{\"text\": \"Claude завершил работу\"}"

Валидация

Проверять что конфиг корректен до применения. Через prompt-хук можно дать LLM оценить изменение.

// PreToolUse + prompt "type": "prompt", "prompt": "Проверь что settings.json\nостаётся валидным JSON после этого изменения"

Автотесты

Прогонять тесты после изменения файлов. Если тесты упали -- Claude сразу узнает.

// PostToolUse matcher="Write|Edit" matcher на .py файлы "command": "cd /path/to/project && pytest tests/ -q 2>/dev/null || true"

⚠ Ограничения и ловушки

1. Не могут писать в чат напрямую

Хуки не выводят текст пользователю в чат. Единственный способ -- через systemMessage в ответе, но это не всегда работает как ожидаешь. Для уведомлений лучше HTTP-запросы.

2. Тихое падение

Если в хуке ошибка -- он молча игнорируется. Claude не покажет "хук упал". Поэтому всегда ставь || true в конце команд и проверяй логи вручную.

3. Не все события в агентах

Хуки во фронтматере агента поддерживают не все типы событий. Некоторые триггеры (SessionStart, Stop) работают только на уровне сессии.

4. Matcher -- regex, а не glob

Фильтр matcher использует регулярные выражения. Write|Edit -- это OR-оператор regex. Не Write, Edit и не *.py. Привыкай к regex-синтаксису.

5. Команда получает JSON на stdin

Shell-команда хука получает JSON с данными о событии через stdin. Нужно его парсить. Python json.load(sys.stdin) или jq -- стандартные инструменты.

# Пример stdin для Write: { "tool_input": { "file_path": "/home/user/config.json", "content": "..." } }

Итого: Хуки -- это автоматические shell-команды на события Claude Code. PreToolUse может блокировать, PostToolUse -- доделывать. Настроены в settings.local.json. У нас: бэкапы + автоформатирование. Простой инструмент, который экономит кучу ручной работы.
