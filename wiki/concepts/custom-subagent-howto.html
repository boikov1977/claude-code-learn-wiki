💡 Что такое субагент

Субагент — это специализированный AI-помощник со своей ролью, инструментами и (опционально) своей моделью. Основной Claude Code может вызывать субагента инструментом Agent и делегировать ему задачи.

Аналогия: Если Claude Code — это тимлид, то субагенты — это его команда. Один специализируется на ревью кода, другой — на документации, третий — на тестах. Каждый делает своё дело хорошо.

Субагенты живут в ~/.claude/agents/ как файлы .md с YAML-шапкой (frontmatter).

⚔️ Skill vs Agent: в чём разница

Skill Agent
Файл ~/.claude/skills/имя/SKILL.md ~/.claude/agents/имя.md
Вызов /имя-скилла @имя или инструмент Agent
Контекст Работает в контексте основной сессии Изолированная песочница (новый контекст)
Модель Может отличаться Может отличаться
Инструменты Ограниченные (tools) Ограниченные (tools + disallowedTools)
MCP Нет Да (mcpServers)
Когда использовать Многошаговые инструкции в контексте сессии Специализированный эксперт с изоляцией

Простое правило:

  • Skill = «сделай это, следуя инструкции» (в контексте сессии)
  • Agent = «ты — эксперт по X, решай задачи в своей области» (изолированно)

📁 Структура файла агента

--- name: my-agent description: Краткое описание роли агента tools: - Read - Write disallowedTools: - Bash mcpServers: [] model: claude-sonnet-4-20250514 --- # Системный промпт Ты — эксперт по [области]. Твоя задача: - [что делаешь] - [как делаешь] - [чего не делаешь] Формат ответа: - [какой формат]

Файл кладётся в ~/.claude/agents/имя.md.

⚙️ Frontmatter: все поля

Поле Обязательно? Тип Описание
name Да string Имя агента. Используется для вызова через @имя.
description Да string Описание роли. Помогает Claude выбрать правильного агента.
tools Нет list Какие инструменты доступны. По умолчанию — все.
disallowedTools Нет list Какие инструменты запрещены. Переопределяет tools.
mcpServers Нет list MCP-серверы, доступные агенту. Пустой список [] = изоляция.
model Нет string Модель для агента. По умолчанию — основная модель сессии.

tools vs disallowedTools

tools — белый список (что разрешено). disallowedTools — чёрный список (что запрещено). Если указаны оба — disallowedTools имеет приоритет.

Пример: разрешить всё, кроме Bash:

tools: all disallowedTools: - Bash

⚠️ disallowedTools критичен для безопасности

Если агент не должен запускать команды — обязательно добавь disallowedTools: [Bash]. Иначе агент сможет выполнить любую shell-команду.

📝 Системный промпт

Всё, что идёт после --- — это системный промпт. Это «характер» и «экспертиза» агента.

Что писать в системном промпте

  • Кто ты: «Ты — эксперт по security review»
  • Что делаешь: «Анализируешь код на уязвимости»
  • Как делаешь: «Проверяешь OWASP Top 10, ищешь SQL injection, XSS»
  • Чего не делаешь: «Не пишешь исправления — только отчёт»
  • Формат: «Ответ в формате: Критично / Важно / Рекомендация»
  • Ограничения: «Не трогай файлы в папке migrations»
💡 Совет: Системный промпт для агента — это не то же самое, что промпт для скилла. Агент работает в изолированном контексте: он не видит историю основной сессии. Поэтому промпт должен содержать всю необходимую информацию о роли и задаче.

🔌 MCP-серверы для агента

Агент может подключать внешние MCP-серверы через поле mcpServers.

Изоляция через mcpServers: []

Если указать mcpServers: [] (пустой список) — агент не будет иметь доступа ни к каким MCP-серверам. Это полезно для изоляции: агент работает только со встроенными инструментами.

mcpServers: [] # полная изоляция от MCP

Подключение конкретных серверов

Если агенту нужен доступ к конкретным MCP-серверам — перечисли их:

mcpServers: - poisk-mcp # веб-поиск - context7 # API-документация

👩‍💻 Пример: code-reviewer

Полный пример агента для код-ревью:

--- name: code-reviewer description: Эксперт по код-ревью. Анализирует изменения на баги, проблемы безопасности и архитектуру. tools: - Read - Glob - Grep disallowedTools: - Write - Edit - Bash mcpServers: [] model: claude-sonnet-4-20250514 --- Ты — эксперт по код-ревью. Твоя задача — проанализировать текущие изменения (git diff) и дать структурированный отчёт. ## Что проверять 1. **Баги**: логические ошибки, off-by-one, null-доступ 2. **Безопасность**: SQL injection, XSS, hardcode секретов 3. **Стиль**: нарушения конвенций проекта, дублирование 4. **Архитектура**: нарушение SRP, циклические зависимости ## Формат отчёта Для каждого найденного вопроса: - **Уровень**: Критично / Важно / Рекомендация - **Файл**: путь и строка - **Описание**: что не так - **Рекомендация**: как исправить ## Ограничения - Не изменяй файлы (у тебя нет Write/Edit/Bash) - Только анализ и рекомендации - Не комментируй стиль если нет серьёзных проблем

Сохрани как ~/.claude/agents/code-reviewer.md.

👆 Как вызвать

Есть два способа:

1. Через @имя

@code-reviewer проанализируй текущий git diff

2. Через инструмент Agent

Claude сам вызовет агента инструментом Agent, когда решит что задача подходит для специалиста. Ты можешь попросить явно: «Вызови code-reviewer для проверки изменений в auth.py»

Проверка: что агент доступен

Набери @ в чате — Claude покажет список доступных агентов. Твой субагент будет в списке, если файл лежит в правильной папке.

💡 Важно: Субагент работает в изолированном контексте. Он не видит историю твоего чата с Claude. Поэтому в вызове нужно явно указать: что анализировать, где файлы, какая задача.

💡 Советы

Одна роль — один агент

Не пытайся сделать «универсального агента». Лучше 3 специалиста: code-reviewer, security-auditor, doc-writer. Каждый делает своё дело хорошо.

Изоляция через mcpServers: []

Для агентов, которые только анализируют — отключи MCP. Это гарантия безопасности: агент не сможет обратиться к внешним сервисам.

disallowedTools для безопасности

Если агент не должен писать код — отключи Write и Edit. Если не должен запускать команды — отключи Bash.

Тестируй через @имя

Вызови агента через @имя с простой задачей. Если он работает как нужно — можно использовать в реальных сценариях.

Используй разные модели

Для простых задач (ревью стиля) — быстрая модель (Sonnet). Для сложных (архитектурный анализ) — мощная (Opus). Экономия на модели = экономия на токенах.

Итого: Субагент — это файл .md в ~/.claude/agents/ с frontmatter (name, description, tools, disallowedTools, mcpServers, model) и системным промптом. Вызывается через @имя. Работает в изолированном контексте.
