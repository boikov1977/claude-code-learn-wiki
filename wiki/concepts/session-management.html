💬 Что такое сессия?

Когда ты открываешь терминал и пишешь claude — ты создаёшь сессию. Это как разговор: ты задаёшь вопросы, Claude отвечает, вы работаете вместе. Разговор длится, пока ты не закроешь терминал (или пока сессия не истечёт).

Каждая сессия — это отдельный контекст. Все сообщения, все действия, все фоновые задачи — всё это живёт внутри одной сессии. Закрыл терминал — разговор прервался.

Проще простого: Сессия — это «диалоговое окно» с Claude Code. Одно окно = одна сессия. Закрыл окно — сессия прервалась. Но не исчезла — она где-то ещё живёт и может вернуться.

🚀 Создание сессии: claude

Самый простой и очевидный способ:

claude

Это создаёт новую сессию — чистый лист. Claude не помнит ничего из предыдущих разговоров (если ты не настроил память в CLAUDE.md).

Пример

# Утром пришёл на работу, открыл терминал $ claude # Claude приветствует тебя и готов работать # Это новая сессия — чистый старт
💡 Важно: Даже если ты запускаешь claude в том же терминале и в той же папке — это новая сессия. Предыдущая не пропала — она просто в другом «окне».

🔄 Возврат в сессию: --resume

Представь: ты работал над задачей, закрыл ноутбук, поехал домой. На диване открыл терминал и хочешь продолжить именно там, где остановился. Вот тут приходит на помощь --resume.

# Вернуться в прерванную сессию claude --resume # Вернуться в конкретную сессию по ID claude --resume <session-id>

--resume делает вот что:

  • Находит последнюю прерванную сессию
  • Восстанавливает всё состояние — историю сообщений, контекст, фоновые задачи
  • Ты продолжаешь ровно с того места, где остановился

Что восстанавливается

Элемент Восстанавливается?
История сообщений Да, полностью
Контекст (что Claude «помнит») Да
Фоновые задачи Зависит от ситуации
Файлы на диске Остаются как есть
CLAUDE.md / память Да, подхватывается

➡️ Новая сессия с контекстом: --continue

А вот другая ситуация: контекст разросся до огромных размеров, Claude начал тупить, отвечает медленно и теряет нить. Ты хочешь начать свежую сессию, но при этом не потерять всё, что обсудили.

# Начать новую сессию, подхватив контекст предыдущей claude --continue

--continue делает следующее:

  • Берёт контекст (содержание) предыдущей сессии
  • Создаёт новую сессию с этим контекстом
  • Старая сессия остаётся жить (но уже не активна)

Аналогия из жизни

Представь большую доску, на которой ты рисовал схему. Доска уже забита — негде писать. Ты фотографируешь её, стираешь всё и начинаешь рисовать заново, глядя на фотографию. Новая доска — чистая, но ты не потерял информацию.

Это два самых частых путаницы. Разберём наглядно.

--resume

  • Та же сессия
  • Тот же session ID
  • Тот же стейт и контекст
  • Фоновые задачи могут быть активны
  • Продолжаешь ровно откуда ушёл
claude --resume

--continue

  • Новая сессия
  • Новый session ID
  • Контекст передан из старой
  • Старая сессия остаётся жить
  • Свежий старт с багажом знаний
claude --continue
💡 Правило большого пальца: Хочешь продолжить — используй --resume. Хочешь начать заново, но не терять контекст — используй --continue.

📁 Где живут сессии и сколько

Все сессии хранятся локально на твоём компьютере:

~/.claude/sessions/

Каждая сессия — это отдельная папка с идентификатором. Внутри — история сообщений, контекст, метаданные. Claude Code хранит их в формате JSON.

TTL (time-to-live)

Сессии не живут вечно. У каждой сессии есть TTL — время жизни. После истечения TTL сессия автоматически удаляется.

Сколько живут сессии

Параметр Значение
Хранение на диске ~/.claude/sessions/
TTL (время жизни) Зависит от настроек, обычно несколько дней
После истечения TTL Сессия удаляется, вернуться нельзя
Ручное удаление Можно удалить вручную из ~/.claude/sessions/

Шаг за шагом:

  1. Создание. Ты пишешь claude — появляется новая сессия.
  2. Активная фаза. Ты общаешься с Claude, он работает, выполняет задачи.
  3. Прерывание. Закрыл терминал. Сессия не умирает сразу — она «висит».
  4. Ожидание. Сессия ждёт:maybe ты вернёшься через --resume.
  5. Восстановление (опционально). Ты пишешь claude --resume — и ты обратно.
  6. Удаление. Прошло время (TTL истёк) — сессия удаляется навсегда.

🎯 Практические сценарии

🏠 Работаешь в офисе → дома

Утром в офисе разбирал баг в auth-модуле. Закрыл ноутбук, поехал домой. На диване открыл терминал:

claude --resume

И ты обратно в той же сессии — Claude помнит весь баг и продолжает с того места.

📈 Контекст разросся

Три часа кодишь большую фичу. Контекст заполнился на 200%, Claude начал отвечать медленно. Начинаешь чистую сессию, но с багажем:

claude --continue

Новая сессия подхватила контекст предыдущей и работает быстрее.

⏳ Фоновая задача отработала

Запустил фоновую задачу --bg, закрыл терминал уходя на обед. Вернулся — задача отработала:

claude --resume

Сессия восстановилась, результат фоновой задачи доступен в истории.

🌙 Вечер → утро

Вечером начал исследовать архитектуру проекта. Уснул. Утром продолжил:

claude --resume

Пока TTL не истёк — ты в той же сессии со всей вчерашней историей.

Когда --resume не поможет

  • TTL истёк — сессия удалена, вернуть нельзя. Придётся начинать с нуля.
  • Ручное удаление — если ты сам удалил папку ~/.claude/sessions/.
  • Сессия с фоновыми задачами — фоновые задачи могут не восстановиться (подробнее в статье про Background Tasks).

Итого: Сессия — это разговор с Claude Code. claude создаёт новую сессию. --resume возвращает в прерванную (та же самая сессия). --continue начинает новую сессию с контекстом старой. Сессии хранятся в ~/.claude/sessions/ и умирают после TTL. Закрыл терминал — не переживай, можно вернуться. 💾
