Расхождения

Вопрос 🚀 Innovator + ⚖️ Pragmatist 🛡️ Conservative Статус
Интерактив vs Чеклист? Интерактивный диалог: вопросы в чат с вариантами ответов, разработчик отвечает последовательно Статический чеклист: 3 вопроса с готовыми ответами, подсветка слабых мест. Без диалога. РАСХОЖДЕНИЕ

Аргументы по расхождению:
За интерактив: пауза для рефлексии, динамические вопросы, формат "с рекомендованным ответом" снижает трение.
За чеклист: тот же эффект без context overhead, нет риска "прожать enter", статика надёжнее на слабых моделях.

7. Оставшиеся риски (позиция Conservative)

  • False sense of coverage — список вопросов предсказуем, разработчик выучит "правильные" ответы. Скилл превращается в ритуал.
  • Context window overhead — 2-5K токенов на каждый запуск, даже если вопросы не дали пользы. На DeepSeek-прокси idle-токены жгут бюджет.
  • Модель не гарантирует глубину — на прокси качество вопросов может быть ниже ожидаемого.
  • Кто решает, когда grill включать? — триггер по complexity предполагает, что агент корректно оценивает сложность. Это не гарантировано.

8. Рекомендация по внедрению

На основе консенсуса:

  • Добавить секцию в openspec-manager SKILL.md (не отдельный файл).
  • Триггер: complexity >= medium && risk >= medium.
  • Skip-флаг: skip-grill: true в change-файле.
  • 2-3 вопроса с вариантами ответов.
  • Вопросы до генерации proposal (не в $BeforeCode).
  • Cost: 10-15 строк. Time: 15 минут.
  • Rollback: удаление секции без последствий.

Открытым остаётся вопрос интерактив vs чеклист. Рекомендуется начать с интерактива (Innovator + Pragmatist), через месяц оценить usage и при необходимости перейти на чеклист (Conservative).
