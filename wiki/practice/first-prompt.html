🎯 Как формулировать промпт

Claude Code работает лучше всего, когда ты даёшь конкретную задачу. Не «сделай что-нибудь полезное», а «напиши функцию, которая делает X, с тестом Y».

Формула хорошего промпта:

Что сделать + Где + Детали

«Напиши функцию parse_config(path: str) → dict в файле utils.py, которая читает YAML-файл и возвращает словарь. Обработай ошибки: если файла нет — вернуть пустой dict.»

Ключевые принципы:

  • Конкретика: Имя функции, файл, поведение при ошибках
  • Границы: Что именно нужно сделать и что не нужно
  • Контекст: Какой язык, фреймворк, стиль
  • Критерий готовности: «Должен пройти тесты», «Должен работать с ...»

👀 Что увидишь в ответе

Когда ты отправляешь промпт, Claude Code проходит три фазы:

1

Анализ

Claude читает файлы проекта, понимает контекст, определяет структуру. Ты видишь сообщения вроде «Читаю utils.py...», «Проверяю зависимости...».

2

План

Claude описывает, что собирается сделать: «Создам функцию в utils.py, напишу тест в test_utils.py, запущу pytest». Это можно одобрить или поправить.

3

Код

Claude применяет изменения: создаёт/редактирует файлы, запускает команды. Ты видишь каждый шаг: какой файл изменяется, какая команда запускается.

Важно: Claude не делает ничего скрытно. Каждое действие видно в интерфейсе: какой инструмент вызывается, какой файл меняется, какая команда запускается. Ты контролируешь весь процесс.

✅ Approve / Reject

По умолчанию Claude Code спрашивает разрешения перед опасными действиями: запись файлов, запуск команд, удаление. Ты видишь промпт:

Claude wants to Write to utils.py: --- (предпросмотр изменений) --- Allow? [Y/n/quit]
Действие Клавиша Что происходит
Approve Y или Enter Действие выполняется. Claude продолжает цикл.
Reject n Действие отменяется. Claude ищет альтернативу или спрашивает уточнение.
Quit q Вся сессия останавливается. Внесённые изменения сохраняются.

Approve для Read-only — автоматически

Операции чтения (Read, Glob, Grep) проходят без вопросов. Это безопасно: Claude просто смотрит на файлы. Подтверждение нужно только для Write, Edit и Bash.

Auto-approve настраивается

Если тебя достали подтверждения — настрой auto-approve для конкретных инструментов. Подробнее — в статье Permissions.

👩‍💻 Пример: скрипт на Python

Полный пример: от промпта до готового результата.

Промпт

напиши Python-скрипт для подсчёта строк в файле. На входе — путь к файлу. На выходе — количество строк. Добавь обработку ошибок: если файл не найден — вывести сообщение. Напиши тест с использованием pytest.

Что происходит

# Claude анализирует проект... # Читает существующие файлы # Определяет: Python проект, pytest для тестов # Claude создаёт функцию: # utils.py def count_lines(filepath: str) -> int: """Подсчитывает количество строк в файле.""" try: with open(filepath, 'r') as f: return sum(1 for _ in f) except FileNotFoundError: print(f"Файл не найден: {filepath}") return 0 # Claude создаёт тест: # test_utils.py from utils import count_lines import tempfile, os def test_count_lines(): with tempfile.NamedTemporaryFile(mode='w', suffix='.txt', delete=False) as f: f.write("line1\nline2\nline3\n") path = f.name assert count_lines(path) == 3 os.unlink(path) def test_file_not_found(): assert count_lines("nonexistent.txt") == 0 # Claude запускает тесты: $ pytest test_utils.py -v test_utils.py::test_count_lines PASSED test_utils.py::test_file_not_found PASSED

Результат: Два файла созданы, тесты прошли. Claude отвечает: «Функция count_lines готова в utils.py. Тесты в test_utils.py проходят. Функция обрабатывает случай отсутствия файла.»

⚔️ Плохой vs хороший промпт

❌ Плохой промпт ✅ Хороший промпт
сделай что-нибудь полезное создай функцию для валидации email в utils.py с тестом
почини баг в файле auth.py функция login падает с TypeError когда password=None
напиши документацию обнови docstring для функции parse_config: добавь описание параметров и примеры
оптимизируй код в functions.py функция find_users делает N+1 запрос к БД. Рефакторни на batch-запрос
добавь тесты напиши тесты для edge cases в utils.py: пустой ввод, None, очень длинная строка
💡 Правило: Если ты не можешь написать конкретный промпт — значит, ты сам не до конца понимаешь задачу. Сначала сформулируй для себя: «что именно должно произойти?» — и только потом пиши Claude.

💡 Советы

1. Начни с простой задачи

Первый промпт не должен быть сложным. Попробуй что-то тривиальное: «напиши функцию hello world с тестом». Цель — увидеть как работает агентный цикл, а не решить реальную задачу.

2. Не бойся отклонять

Если Claude делает не то, что ты хотел — отклоняй (n) и уточняй промпт. Claude запоминает контекст текущей сессии и адаптируется.

3. Следи за контекстом

Если Claude начинает повторяться или «забывать» ранние шаги — контекст заполнился. Вызови /compact или начни новую сессию.

4. Используй CLAUDE.md

Для проекта создай CLAUDE.md с инструкциями. Тогда Claude будет знать стек, конвенции и ограничения — и промпты можно писать короче.

Итого: Хороший промпт = конкретная задача + где делать + детали. Claude проходит три фазы: анализ → план → код. Ты контролируешь через approve/reject. Начни с простой задачи и набирайся опыта.
