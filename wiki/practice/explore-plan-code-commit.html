🔍 Explore: изучить код

Прежде чем что-то менять — пойми, что уже есть. Explore — это фаза исследования без изменений.

Что Claude делает на этом этапе

  • Смотрит структуру проекта: ls, Glob
  • Ищет связанные файлы: Grep по ключевым словам
  • Читает файлы: Read по одному или несколько
  • Анализирует зависимости и паттерны
  • Определяет, какие файлы нужно будет менять
# Типичные промпты для Explore: Изучи проект: покажи структуру и ключевые файлы Найди где реализована форма регистрации — покажи файлы и их роль Покажи как работает аутентификация — от входа до проверки токена
💡 Совет: Explore полезен даже когда ты «знаешь проект». Claude может найти файлы, которые ты забыл, или показать связи, которые ты не видел.

📐 Plan: составить план

Plan Mode — это режим, в котором Claude только читает и анализирует, не меняя файлы. Он составляет план: что, где, как менять.

Как войти в Plan Mode

В интерфейсе Claude Code нажми Shift+Tab или напиши включи Plan Mode. Claude переключится в режим «только чтение».

План будет состоять из:

  • Какие файлы нужно создать/изменить
  • Какой подход будет использован
  • Какие тесты нужно написать
  • Какие могут быть побочные эффекты

EnterPlanMode / ExitPlanMode

Это внутренние инструменты Claude Code:

  • EnterPlanMode — Claude входит в режим планирования (read-only). Все write-инструменты отключены.
  • ExitPlanMode — Claude выходит из режима планирования и переходит к выполнению.

✏️ Code: реализовать

После утверждения плана Claude переходит к написанию кода. Это фаза активных изменений: создание файлов, редактирование, запуск команд.

Task Tool для отслеживания

Для задач, которые состоят из нескольких шагов, Claude использует Task Tool — внутренний todo-лист. Он создаёт список задач и отмечает выполненные.

  • Создаёт список: «Создать компонент кнопки», «Добавить обработчик», «Написать тест»
  • Отмечает задачу как «в процессе» когда начинает работу
  • Отмечает как «выполнено» когда закончил
  • Ты видишь прогресс в интерфейсе

Подробнее — в статье Task Tool.

Что происходит во время Code

  • Claude создаёт/редактирует файлы по плану
  • Запускает тесты после каждого изменения
  • Если тесты упали — читает ошибку и исправляет
  • Повторяет, пока не добьётся успеха
  • Каждое действие видно и требует approve (если не настроен auto-approve)

✅ Commit: закоммитить

Когда все изменения готовы и тесты проходят — Claude делает коммит.

Что делает Claude на этом этапе

  1. git add — добавляет изменённые файлы
  2. git commit — создаёт коммит с описательным сообщением

Сообщение коммита обычно включает:

  • Краткое описание что сделано
  • Ссылку на задачу (если была)
  • Список изменённых файлов
# Типичный коммит от Claude: feat: add reset button to registration form - Add ResetButton component in src/components/ResetButton.tsx - Add handleReset handler in RegistrationForm.tsx - Add tests for reset functionality
💡 Совет: Если хочешь чтобы Claude сделал коммит — скажи явно: «закоммить» или «сделай git commit». Claude не делает коммит автоматически, пока ты не попросишь.

👩‍💻 Пример: кнопка сброса

Полный цикл на примере: «Добавить кнопку сброса в форму регистрации».

Step 1: Explore

Ты: изучи проект и найди где реализована форма регистрации Claude: Читаю структуру проекта... Нашёл: src/components/RegistrationForm.tsx Зависимости: React, Formik, Yup Стили: CSS Modules

Step 2: Plan

Ты: составь план: как добавить кнопку сброса. Не пиши код. Claude: План: 1. Создать компонент ResetButton.tsx 2. Добавить handleReset в RegistrationForm 3. Вставить кнопку в JSX рядом с Submit 4. Добавить тест для handleReset

Step 3: Code

Ты: делай Claude: [Task 1/4] Создаю ResetButton.tsx... [Task 2/4] Добавляю handleReset... [Task 3/4] Вставляю кнопку в JSX... [Task 4/4] Запускаю тесты... ✓ Все тесты прошли

Step 4: Commit

Ты: закоммить Claude: $ git add -A $ git commit -m "feat: add reset button to registration form" [main abc1234] feat: add reset button to registration form

💡 Советы

Не пропускай Explore

Даже если ты уверен, что знаешь проект — дай Claude изучить его. Он может найти файлы, которые ты забыл, или показать неочевидные зависимости.

Plan Mode для крупных задач

Для задач крупнее «напиши одну функцию» — обязательно используй Plan Mode. Это сэкономит время: Claude продумает подход до того, как начнёт писать код.

Task Tool для видимости прогресса

Если задача из нескольких шагов — Claude автоматически создаст todo-лист. Ты видишь прогресс и можешь остановить на любом шаге.

Коммит — отдельный шаг

Claude не коммитит автоматически. Это даёт тебе контроль: можешь проверить diff перед коммитом. Когда готов — скажи «закоммить».

Итого: Explore (изучи) → Plan (спланируй) → Code (реализуй) → Commit (закоммить). Четыре шага для любой задачи от простой до сложной.
