Что это за проект

О чём этот сайт и кому он будет полезен

Этот сайт — живая документация по построению multi-agent системы для разработки на базе Claude Code. Мы не просто написали абстрактную статью — мы собрали команду из 13 субагентов на китайских AI-моделях (DeepSeek, MiMo, MiniMax) и задокументировали каждый шаг.

Здесь ты найдёшь:

  • Архитектуру субагентов: писатель, редактор, кодеры, тестировщик, ревьюер, архитекторы
  • Схему работы литературного отдела: writer → editor → fix → publish
  • Архитектурные дебаты: как мы спорили, и к каким вердиктам пришли
  • Гайды по MCP, OpenCode-прокси, Workflow, Plan mode и другим концепциям

Для кого это

Для разработчиков, которые хотят build из своих Claude Code не просто "помощника в терминале", а полноценную команду AI-агентов. Если ты когда-нибудь думал «а пусть один агент пишет код, второй ревьюит, третий документирует» — ты по адресу.

Частые вопросы

Короткие ответы на то, что обычно спрашивают

Нужна ли подписка Claude Pro?

Нет. Проект построен на OpenCode-прокси, который перенаправляет запросы к DeepSeek, MiMo и MiniMax. Это бесплатные или условно-бесплатные модели.

Можно ли использовать это без китайских моделей?

Да. Архитектура субагентов не привязана к конкретному провайдеру. Если у тебя есть Claude Pro — просто убери прокси и поставь модели sonnet/opus в frontmatter агентов.

Сколько это стоит?

Зависит от используемых моделей. DeepSeek-V4-Flash стоит копейки, MiMo-V2.5 — дёшево для long-context задач. MiniMax-M3 — средняя цена. Если перейти на Claude API — будет дороже, но качественнее.

Это только для кода?

Нет. У нас есть литературный отдел (пишет и редактирует статьи), архитектурный отдел (дебаты и вердикты), исследователь (deep research). Сфера применения — любая работа с текстом.

Как быстро можно собрать такую же команду?

Базовую команду из 5-7 агентов можно настроить за день. Промпты — в файлах, копируй и адаптируй. Полный цикл с архитектурными дебатами и литературным отделом — неделя-две.
