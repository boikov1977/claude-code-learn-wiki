🚀 Что такое Claude Code и зачем субагенты

Коротко: терминальный AI-ассистент, который пишет код и управляет файлами

Claude Code — это терминальный инструмент от Anthropic. Запускаешь в терминале, он видит твой проект, читает файлы, пишет код, делает коммиты, запускает тесты. Работает как AI pair programmer прямо в консоли.

Но один Claude — как один разработчик: хорошо делает одно, но не может одновременно кодить, проводить ревью, документировать и тестировать. Субагенты — это специализированные AI-помощники, каждый со своей ролью. Один пишет бэкенд, другой — фронт, третий — документацию. Основной Claude управляет ими как тимлид.

Что ты получишь через 10 минут

Установленный Claude Code OpenCode-прокси с DeepSeek MCP-поиск poisk-mcp Готовые субагенты

  • Claude Code — терминальный AI-ассистент для кода
  • Субагенты — markdown-файлы с конфигурацией (имя, модель, инструменты) и системным промптом в ~/.claude/agents/
  • OpenCode-прокси — перенаправление запросов от Claude Code к другим моделям (DeepSeek, MiMo, MiniMax)
  • MCP — протокол для подключения внешних инструментов (поиск, файлы, БД)

📦 Установка и запуск Claude Code

npm install, проверка версии, запуск в проекте
1

Установка

Claude Code ставится через npm. Если Node.js уже есть:

# Установка (если не установлен) npm install -g @anthropic-ai/claude-code # Проверить версию claude --version
💡 Если npm не установлен — сначала поставь Node.js: curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash - && sudo apt install -y nodejs
2

Запуск

Перейди в директорию проекта и запусти:

# Перейти в проект cd /path/to/your/project # Запустить Claude Code claude

Откроется интерактивный терминал. Claude увидит структуру проекта и будет готов к работе. Можно сразу задавать вопросы или ставить задачи.

Первый запуск

При первом запуске Claude предложит создать файл CLAUDE.md — это память проекта: контекст, правила, особенности. Соглашайся — потом сможешь отредактировать.

🔌 Подключение OpenCode-прокси

DeepSeek, MiMo и другие модели через единый прокси

Claude Pro/Team подписки нет — напрямую к Anthropic API подключаться не получится. OpenCode-прокси решает эту проблему: он принимает запросы Claude Code и перенаправляет их в другие модели (DeepSeek, MiMo, MiniMax).

1

Установка прокси

# OpenCode-прокси может быть развёрнут как Go-сервис или Docker-контейнер. # Уточни способ установки в документации OpenCode. # После запуска прокси — укажи его в ~/.claude.json:
2

Настройка Claude Code

Добавь прокси в ~/.claude.json:

{ "proxy": "http://localhost:8080" }
3

Доступные модели

Каждая модель оптимизирована под свой тип задач:

Модель Назначение Кто использует
claude-go-deepseek-v4-flash Быстрая, для coding-задач Основной Claude, py-backend-coder, frontend-coder, android-coder
claude-go-deepseek-v4-pro Мощная, для анализа code-reviewer, pragmatist
claude-go-mimo-v2.5 Agentic reasoning, длинный контекст writer-agent, tester-agent
claude-go-minimax-m3 Архитектурные решения innovator
💡 Прокси просто передаёт model ID в OpenCode Go — сам не решает, какую модель использовать. Модель указывается в frontmatter каждого агента.

🔍 Подключение MCP-поиска

poisk-mcp — веб-поиск и чтение страниц для Claude

Встроенный WebSearch Claude Code через OpenCode-прокси не работает (ошибка схемы DeepSeek). Для веб-поиска нужен MCP-сервер — внешний процесс, который Claude вызывает как инструмент.

1

Подключение poisk-mcp

# Добавить MCP-сервер глобально claude mcp add poisk-mcp --stdio \ /home/andrei/mcp/poisk-mcp/venv/bin/python \ /home/andrei/mcp/poisk-mcp/server.py # Проверить, что сервер подключён claude mcp list
2

Основные инструменты

  • search_web — поиск по запросу, возвращает ссылки с описаниями
  • read_webpage — чтение содержимого страницы (HTML → текст)
  • browse_summarize — чтение + автоматическое резюме
  • web_research — глубокое исследование за один вызов

Пример вызова

Claude вызывает search_web("MCP servers 2026") — poisk-mcp ищет в вебе, возвращает список ссылок. Claude читает лучшие через read_webpage и анализирует.

🤖 Первый запуск агента

Субагенты — это markdown-файлы с промптом и конфигурацией

Агенты живут в ~/.claude/agents/ — это markdown-файлы с YAML-frontmatter. В frontmatter указываются имя, описание, модель и доступные инструменты. Claude Code загружает их и вызывает через Agent tool.

1

Посмотри, какие агенты уже есть

ls ~/.claude/agents/

Типичный набор:

  • py-backend-coder — Python и backend
  • frontend-coder — React/TS/Vue
  • code-reviewer — ревью кода, read-only
  • writer-agent — документация
  • tester-agent — тесты
  • explorer — быстрый поиск по коду
  • innovator — архитектор-инноватор
  • pragmatist — архитектор-прагматик
2

Вызов агента

Чтобы вызвать субагента, скажи Claude:

# Прямой вызов — в интерактивном чате Используй агента py-backend-coder Задача: напиши simple HTTP сервер на FastAPI # Или через Agent tool (автоматически) Claude сам решит, какого агента вызвать, исходя из описания задачи

Как это работает внутри

Claude Code читает файл агента, берёт оттуда промпт и frontmatter, создаёт дочерний поток с указанной моделью и инструментами, передаёт задачу. Агент выполняет, возвращает результат. Основной Claude получает ответ и интегрирует его в работу.

⌨️ Пять команд, которые нужно знать

Базовый набор для повседневной работы
Команда Что делает Когда использовать
/init Инициализирует проект: создаёт CLAUDE.md с контекстом Первый запуск в новом проекте
/compact Сжимает контекст сессии Когда контекст переполнен и Claude начинает "забывать"
/context Показывает, сколько контекста занято Чтобы понять, сколько ещё места осталось
claude mcp list Показывает активные MCP-серверы Проверить, подключены ли poisk-mcp и другие
/agent <name> Запускает субагента по имени Когда нужно delegation конкретному агенту
💡 Совет: /compact — первый друг, когда сессия длинная. Claude начнёт повторяться или терять контекст — компактифицируй, и он сожмёт историю до ключевых фактов.

✏️ Как создать своего агента

Копируй существующий, меняй frontmatter и промпт

Самый простой способ — скопировать уже работающего агента и адаптировать:

1

Скопируй шаблон

cp ~/.claude/agents/explorer.md ~/.claude/agents/my-agent.md
2

Отредактируй frontmatter

Frontmatter — это YAML-блок в начале файла между ---:

--- name: my-agent description: Мой кастомный агент для конкретной задачи model: claude-go-deepseek-v4-flash tools: - Read - Write - Bash --- # Промпт агента Ты — агент, который занимается [описание задачи]. ## Правила - Правило 1 - Правило 2
3

Ключевые поля frontmatter

Поле Тип Описание
name string Уникальное имя агента (используется для вызова)
description string Описание роли — Claude читает его, чтобы выбрать агента
model string ID модели: inherit (от родителя) или конкретная модель
tools list Какие инструменты доступны: Read, Write, Bash, Agent, Skill и т.д.

model: inherit

Если указать model: inherit — агент унаследует модель родительского Claude. Это удобно, когда не нужна отдельная модель. Конкретная модель (например, claude-go-mimo-v2.5) нужна, когда задача требует специализации.

Поток данных

  1. Разработчик пишет запрос в терминале
  2. Claude Code получает запрос, анализирует задачу
  3. Если нужен веб-поиск — вызывает MCP-сервер (poisk-mcp)
  4. Если задача delegation — вызывает субагента через Agent tool
  5. Запрос к модели идёт через OpenCode-прокси (DeepSeek/MiMo)
  6. Результат возвращается разработчику в терминал
